# <a name="the-sample-itinerary-scenarios"></a>Los escenarios de itinerarios de ejemplo
En la tabla siguiente enumera todos los archivos de itinerario predefinidos incluidos con el ejemplo de itinerario en rampa. Estos se encuentran en la carpeta \Source\Samples\Itinerary\Itineraries.  
  
|Nombre de archivo|Description|  
|---------------|-----------------|  
|Disassembler_OneWay-MessageTransform-MessageRouting-MessgeSendPort.xml|Este itinerario muestra un mensaje predefinido flujo que, cuando envía el itinerario en rampa indica a Microsoft BizTalk para ejecutar una asignación con respecto al mensaje enviado y, a continuación, enrutar el mensaje transformado a n; número de ubicaciones a través de una dinámica personalizada puerto de envío. El número de resoluciones asociada con el servicio determina el número de rutas. Todos transformación y el enrutamiento se produce en el nivel de mensajería de BizTalk, con un servicio de mensajería de enrutamiento opcional compatible con los componentes de canalización de distribuidor de ESB. Las resoluciones asociadas con el servicio en el itinerario determinan las ubicaciones de enrutamientos. Ejecuta este itinerario genera los siguientes mensajes:|  
||2 mensajes en la carpeta \Source\Samples\DynamicResolution\Test\Filedrop\Out|  
||1 mensaje en la carpeta FTP: FTP://localhost/out/%MessageID%.xml|  
||1 mensaje en la cola de MQSeries: MQS://localhost/ESB. USO DE DEP. Sample.QueueManager/TEST. OUT/%MessageID.Xml|

|OneWay MessgeSendPort.xml|Este itinerario muestra un flujo de mensajes predefinido que, cuando envía el itinerario en rampa indica a BizTalk para enrutar el mensaje a una única ubicación a través de una dinámica personalizada puerto de envío.
