# <a name="proactivity-in-biztalk-server"></a>Proactividad en BizTalk Server
Artículo técnico de BizTalk  
  
 **Proactividad en BizTalk Server**  
  
 **Escritor:** Tord contentos Nordahl (Bouvet AS)  
  
 **Revisores técnicos:** Sandro Pereira (Devscope), Jan Steef Wiggers (Motion10), Erik Thue (Bouvet AS), Alexander Thue (Bouvet AS), Saravana Kumar (BizTalk360), Mandi Ohlinger (Microsoft)  
  
 **Fecha de publicación:** julio de 2013  
  
 **Se aplica a:** servidor BizTalk Server  
  
 **Resumen:** las notas del producto se aplica desde BizTalk Server 2004 a BizTalk Server 2013 entornos.  El objetivo es proporcionar más información sobre cómo ser proactivo y resolver o evitar problemas potenciales antes de que ocurran. Se aplica a todos los tipos de usuarios de BizTalk, incluidos los arquitectos, desarrolladores y administradores. No todos los escenarios descritos pueden producirse en su entorno y pueden producirse algunos escenarios no mencionados en este documento.  
  
 Debido a la naturaleza y la importancia de BizTalk Server, configuración y prácticas recomendadas pueden variar entorno; incluida la configuración de red, diseño de arquitectura, versiones, flujo de mensajes y recursos. Estas notas del producto se describe la configuración y valores recomendados.
