# <a name="phase-2-planning-the-assessment"></a>Fase 2: Planear la evaluación
La fase de planificación de una evaluación de rendimiento en gran medida consiste en establecer los hitos específicos para la finalización de las entregas identificadas durante la fase de ámbito. La fase de planificación es "when" a la fase de ámbito del "qué". Software de terceros y logística físico es "cómo" y "donde" aspectos que deben considerarse también durante la fase de planeación como estos aspectos pueden requerir el tiempo adicional para implementar.  
  
 En este tema se describe los distintos aspectos de la fase de planificación de una evaluación de rendimiento.  
  
## <a name="solution-milestones-timeline"></a>Escala de tiempo de los hitos de solución  
 Es importante contar con un registro claro y conciso de los hitos de la evaluación del rendimiento. Establecer los hitos específicos, aumentará la probabilidad de que la solución se completó de manera oportuna. Considere la posibilidad de crear una escala de tiempo de Visio para representar visualmente las fechas, hitos y las tareas asociadas con la evaluación del rendimiento. Mantener la escala de tiempo lo suficientemente sencilla para que sea fácil de comprender aún lo suficientemente detallado para establecer una comunicación eficaz hitos. Los hitos reales se deben acordar entre todas las partes interesadas y revisarse periódicamente. Una escala de tiempo de Visio de ejemplo se muestra a continuación:

## <a name="non-microsoft-software-considerations"></a>Consideraciones de software que no sean de Microsoft  
 Es recomendable tener muy claro desde el principio sobre los cuales las partes interesadas proporcionarán la experiencia necesaria para cualquier software de terceros que se usará durante la evaluación del rendimiento. Tenga en cuenta lo siguiente al software de Microsoft no se utilizarán con la solución:  
  
-   Determinar cómo el software o hardware necesario obtenido.  
  
-   Planear la capacidad y ajuste de tamaño para asegurarse de que el software no sean de Microsoft no es un cuello de botella en la solución.  
  
-   Determinar un plan de acción para la instalación de software de terceros necesarias.  
  
-   Crear un plan de acción para configurar y optimizar el software de terceros necesarias.  
  
## <a name="physical-space-and-other-logistics"></a>Espacio físico y otro logística  
 Espacio físico es un detalle importante a tener en cuenta durante la fase de planeación. Para llevar a cabo un laboratorio, deben existir donde puede tener acceso y los participantes requerirá espacio para trabajar y mantener conversaciones en equipos. Los participantes también se beneficiarán de espacio adicional que se pueden mantener conferencias telefónicas en un foro público menor.  
  
 Uno de los aspectos más importantes del entorno físico del laboratorio de pruebas es crear un entorno donde todos los participantes pueden ver cómodamente el mismo conjunto de pantallas. Esto puede realizarse mediante proyectores a través del cual el equipo puede ver sesiones de escritorio remoto que se usa durante la práctica o sombrear sesiones mediante la característica de "sombra" de escritorio remoto.  
  
 Contrataciones de evaluación del rendimiento pueden producir problemas inesperados, que solo se puede agravarse por el hecho de que la contratación suele ser según una programación estrecha. Debido a los días laborables a veces muy largos que deben cumplir los hitos, será importante para el éxito de la contratación logística espacio físico.  
  
 En resumen, los siguientes factores deben tener en cuenta al planear el espacio físico necesario para la evaluación del rendimiento:  
  
-   Equipos del laboratorio deben ser accesibles para los miembros del equipo.  
  
-   Los miembros del equipo necesitará espacio para albergar las discusiones durante la práctica.  
  
-   Los miembros del equipo deben tener acceso a medios visuales que permite a todos los miembros ver el mismo conjunto de pantallas, normalmente ya sea un proyector vinculado a una pantalla de escritorio remota o mediante la característica de "sombra" de escritorio remoto.  
  
-   Los miembros del equipo deben tener acceso a un "área de pizarra" que se usará para facilitar las discusiones durante la práctica.  
  
-   Los miembros del equipo deben tener acceso al espacio donde conferencias de teléfono se pueden guardar en un foro público menor.  
  
-   Si es posible, el área de laboratorio debe ser control extendido del horario de climatización, esto se alojará acceso horas si es necesario.  
  
-   Todos los usuarios probablemente todavía, necesitará para comunicarse con sus trabajos de día, así que debe dejar suficiente espacio de tabla para los equipos cliente y sus equipos portátiles, 2 para cada participante.  
  
-   Asegúrese de que están disponibles en el laboratorio suficiente regletas de enchufes y conexiones de red. Si se usa inalámbrica, organizar el acceso de invitados para que los participantes.  
  
-   Asegúrese de que el entorno de laboratorio será seguro durante horas de poca actividad.
