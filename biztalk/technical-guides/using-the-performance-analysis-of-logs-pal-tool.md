# <a name="using-the-performance-analysis-of-logs-pal-tool"></a>Usar el análisis de rendimiento de la herramienta de registros (PAL)
La herramienta PAL (análisis de registros de rendimiento) lee en un registro de contador del monitor de rendimiento (cualquier formato conocido) y analiza con umbrales complejos, pero conocidos (siempre). La herramienta genera un informe basada en HTML que gráficamente importantes de rendimiento de los gráficos de contadores y genera alertas cuando se superan los umbrales. Los umbrales originalmente se basan en los umbrales definidos por los equipos de producto de Microsoft, incluidos [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)], y los miembros de Microsoft admiten. Esta herramienta no es un sustituto de análisis de rendimiento tradicionales, pero automatiza el análisis de rendimiento de registros de contador suficientemente para ayudar a ahorrar tiempo. La herramienta PAL:  

- Analiza los registros de contador de rendimiento para los umbrales  

- Es útil para los registros de Perfmon grandes  

- Identifica [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] y sistema operativo cuellos de botella de contador de rendimiento mediante el análisis de los umbrales de  

- Es extensible para realizar el análisis en todos los contadores de rendimiento  

- Puede usarse para ayudar a escribir su propio contador  

  Está disponible como descarga gratuita en PAL [ http://go.microsoft.com/fwlink/?linkid=98098 ](http://go.microsoft.com/fwlink/?linkid=98098). Requiere Microsoft Log Parser. Log Parser es una herramienta eficaz y versátil que ofrece acceso universal de consulta a datos basados en texto tales como archivos de registro, archivos XML y archivos CSV, así como orígenes de datos clave en el sistema operativo de Windows, como el registro de eventos, el registro, el sistema de archivos y activo Servicio de directorio Directory®. Es posible que desee usar esta herramienta para consultar una cantidad significativa de información de registro. Puede descargar la herramienta Log Parser en [ http://go.microsoft.com/fwlink/?linkid=85574 ](http://go.microsoft.com/fwlink/?linkid=85574).  

## <a name="using-pal-with-performance-counter-logs-in-different-languages"></a>Mediante la PAL con el contador de rendimiento de los registros en distintos idiomas  
 La herramienta PAL analiza los registros de contador de rendimiento sólo en idioma inglés. Para usar la herramienta PAL con registros de contador de rendimiento en otros lenguajes, primero debe traducir los registros para el idioma inglés. Puede usar el **traductor de registro Perfmon** herramienta disponible en [ http://go.microsoft.com/fwlink/?LinkId=158802 ](http://go.microsoft.com/fwlink/?LinkId=158802) para traducir los archivos de registro de contador de rendimiento original en inglés.

## <a name="available-memory-analysis"></a>Análisis de la memoria disponible

## <a name="processor-queue-length-analysis"></a>Análisis de longitud de cola de procesador  
 "Longitud de cola de procesador" es el número de subprocesos en la cola del procesador. Hay una única cola para el tiempo de procesador incluso en equipos con varios procesadores. Por lo tanto, si un equipo tiene varios procesadores, necesita dividir este valor por el número de procesadores que atienden la carga de trabajo.  Este análisis determina si la longitud de cola promedio del procesador supera el número de procesadores. Si es así, esto podría indicar un cuello de botella de procesador. La cola del procesador es la colección de subprocesos que están listos, pero no puede ser ejecutado por el procesador porque otro subproceso activo actualmente se está ejecutando.  Puede usar este contador junto con el "procesador\\% Processor Time" contador para determinar si la aplicación puede beneficiarse de más CPU. ## <a name="privileged-mode-cpu-analysis"></a>Análisis del modo con privilegios de CPU  
 Este contador indica el porcentaje de tiempo que un subproceso se ejecuta en modo privilegiado. Cuando la aplicación llama a funciones de sistema operativo (por ejemplo, para realizar operaciones de E/S de archivo o de red o asignar memoria), estas funciones del sistema operativo se ejecutan en modo privilegiado.  

 Modo privilegiado alta CPU indica que el equipo dedica demasiado tiempo en los trabajos de E/S del sistema frente a real (modo de usuario). "% De tiempo privilegiado" es el porcentaje de tiempo transcurrido que los subprocesos del proceso han dedicado a ejecutar código en modo privilegiado.  Cuando se llama a un servicio de sistema de Windows, el servicio a menudo se ejecutará en modo privilegiado para obtener acceso a datos privados del sistema. Dichos datos están protegidos frente al acceso por subprocesos que se ejecutan en modo de usuario. Las llamadas al sistema pueden ser explícitas o implícitas, como errores de página o interrupciones. A diferencia de algunos sistemas operativos anteriores, Windows delimita los procesos para la protección del subsistema además de la protección tradicional mediante los modos con privilegios de usuario y. Algún trabajo realizado por Windows en nombre de la aplicación puede aparecer en otros procesos del subsistema además del tiempo privilegiado en el proceso.  

 Este análisis se comprobación para ver si el modo privilegiado CPU está consumiendo más de un 30 por ciento del total de CPU. Si es así, a continuación, es probable que el consumo de CPU causado por otro cuello de botella que no sea el procesador, como la red, memoria o E/S de disco. Utilizar en correlación con el % de tiempo de interrupción y el cambio de contexto alto los análisis.  

## <a name="high-context-switching-analysis"></a>Análisis de conmutación de contexto alto  
 Se produce un cambio de contexto cuando un subproceso con mayor prioridad anticipa un subproceso de menor prioridad que se está ejecutando actualmente o cuando un subproceso se bloquea una prioridad alta. Los niveles altos de cambio de contexto pueden producirse cuando varios subprocesos comparten el mismo nivel de prioridad. A menudo, esto indica que hay demasiados subprocesos compiten por los procesadores en el sistema. Si no ve mucho uso de procesador y vea niveles muy bajos de cambio de contexto, podría indicar que los subprocesos se bloquean.  

 Cambio de contexto alta solo se debe analizar al modo privilegiado CPU y el total de la CPU es alto. Como norma general, las tasas de cambio de contexto de menos de 5.000 por segundo por cada procesador no son vale la pena preocuparse. Si las tasas de cambio de contexto supera los 15 000 por segundo por cada procesador, hay una restricción.  

 Este análisis busca alta CPU, CPU de modo privilegiado alta y alta (más de 5.000 por procesador) del sistema de cambios de contexto por segundo produciendo todas al mismo tiempo. Si se produce el cambio de contexto alto, a continuación, reduzca el número de subprocesos y procesos que se ejecutan en el sistema.

## <a name="biztalkmessage-agent-process-memory-usage-mb-throttling-threshold-analysis"></a>Umbral de limitación BizTalk agente proceso memoria uso (MB)  
 Este es el umbral actual para el uso de memoria del proceso actual (MB). El umbral se puede ajustar dinámicamente según la cantidad de memoria disponible para este proceso y su patrón de consumo de memoria real. Limitación de memoria del proceso de BizTalk puede producirse si el lote a publicar tiene requisitos de memoria elevados o si hay demasiados subprocesos están procesando mensajes. Si parece que el sistema sobrelimitado, considere la posibilidad de aumentar el valor asociado con el umbral de uso de memoria de proceso para el host y compruebe que la instancia de host no genera un error de "memoria insuficiente". Si se produce un error de "memoria insuficiente" al aumentar el umbral de uso de memoria de proceso, considere la posibilidad de reducir los valores para el tamaño de la cola de mensajes interna y mensajes por los umbrales de CPU en curso. Esta estrategia es especialmente importante en escenarios de procesamiento de mensajes de gran tamaño.  

 Si el servidor BizTalk server con regularidad quedarse sin memoria virtual, a continuación, considere la posibilidad de BizTalk Server de 64 bits. Cada proceso en los servidores de 64 bits puede tratar de hasta 4 TB de memoria virtual frente a de los 2 GB 32 bits. En general, BizTalk de 64 bits y 64 bits de SQL Server se recomienda encarecidamente. Consulte la referencia de "compatibilidad con BizTalk Server de 64 bits" para obtener más información. Este análisis comprueba si la limitación de memoria de proceso está establecida en un valor no predeterminado.
