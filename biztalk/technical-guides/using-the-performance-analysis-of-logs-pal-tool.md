# <a name="using-the-performance-analysis-of-logs-pal-tool"></a>Usar el análisis de rendimiento de la herramienta de registros (PAL)
La herramienta PAL (análisis de registros de rendimiento) lee en un registro de contador del monitor de rendimiento (cualquier formato conocido) y analiza mediante umbrales complejos, pero conocidos (proporcionados). La herramienta genera un informe basada en HTML que gráficamente importantes de rendimiento de los gráficos de los contadores y genera alertas cuando se superan umbrales. Los umbrales originalmente basados en umbrales definidos por los equipos de producto de Microsoft, incluidos los [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)], y los miembros de Microsoft admiten. Esta herramienta no es un sustituto de análisis de rendimiento tradicionales, pero automatiza el análisis de rendimiento de registros de contador suficiente para ayudar a ahorrar tiempo. La herramienta PAL:  
  
-   Analiza los registros de contador de rendimiento de umbrales  
  
-   Es útil para los registros de Perfmon grandes  
  
-   Identifica [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] y cuellos de botella del contador de rendimiento de sistema operativo mediante el análisis de umbrales  
  
-   Es extensible para realizar el análisis en todos los contadores de rendimiento  
  
-   Puede usarse para ayudar a escribir sus propios contadores  
  
 Está disponible como descarga gratuita en PAL [ http://go.microsoft.com/fwlink/?linkid=98098 ](http://go.microsoft.com/fwlink/?linkid=98098). Requiere analizador del registro de Microsoft. Analizador del registro es una herramienta eficaz y versátil que proporciona acceso universal de consulta a datos basados en texto, como archivos de registro, archivos XML y archivos CSV, así como orígenes de datos clave en el sistema operativo de Windows, como el registro de eventos, el registro, el sistema de archivos y Active Servicio de directorio de Directory®. Puede usar esta herramienta para consultar una cantidad significativa de información de registro. Puede descargar la herramienta de analizador del registro en [ http://go.microsoft.com/fwlink/?linkid=85574 ](http://go.microsoft.com/fwlink/?linkid=85574).  
  
## <a name="using-pal-with-performance-counter-logs-in-different-languages"></a>Uso de PAL con el contador de rendimiento de registros en distintos idiomas  
 La herramienta PAL analiza los registros de contador de rendimiento sólo en inglés. Para usar la herramienta PAL con registros de contador de rendimiento en otros lenguajes, primero debe traducir los registros para el idioma inglés. Puede usar el **Perfmon registro traductor** herramienta disponible en [ http://go.microsoft.com/fwlink/?LinkId=158802 ](http://go.microsoft.com/fwlink/?LinkId=158802) para traducir los archivos de registro de contador de rendimiento original en inglés.

|9|Limitación debido a número excesivo de subprocesos|

## <a name="biztalk-message-agent-in-process-message-count-throttling-analysis"></a>Número de mensajes de BizTalk para Message agente en curso limitación Analysis  
 Este es el número de mensajes simultáneos que está procesando la clase de servicio. La configuración de "mensajes en curso por CPU" en la configuración de la limitación de Host es el número máximo de mensajes entregados al Gestor extremo (EPM) o XLANG que no se ha procesado. Esto no incluye los mensajes recuperados de la base de datos pero que aún esperan para la entrega en la cola en memoria. Puede supervisar el número de mensajes en proceso mediante el contador de rendimiento de recuento de mensajes en curso en la categoría de objeto de rendimiento BizTalk: agente. Este parámetro proporciona una sugerencia al mecanismo de limitación para que tenga en cuenta las condiciones de limitación. El umbral real se ajusta automáticamente. Puede comprobar el umbral real al supervisar el contador de rendimiento de recuento de mensajes en curso.  
  
 Para escenarios de mensajes de gran tamaño (donde el tamaño promedio del mensaje es alta, o el procesamiento de mensajes puede requerir una gran cantidad de mensajes), este parámetro puede establecerse en un valor menor. Un escenario de mensajes de gran tamaño se indica en caso de limitación basada en memoria con demasiada frecuencia, y si el umbral de memoria se ajusta de forma automática a un valor notablemente bajo. Dicho comportamiento indica que el transporte de salida debe procesar menos mensajes simultáneamente para evitar un uso excesivo de la memoria. Asimismo, para los escenarios en los que el adaptador es más eficaz cuando procesa pocos mensajes a la vez (por ejemplo, cuando se envía a un servidor que limita las conexiones simultáneas), este parámetro se puede ajustar con un valor inferior al predeterminado. Este análisis comprueba el elevado número de mensajes en proceso para determinar si es mayor que 80 por ciento de su configuración de limitación en el mismo nombre, es probable que lo que indica una condición de limitación.

## <a name="biztalk-message-agent-process-memory-usage-mb-throttling-analysis"></a>Mensaje de BizTalk agente proceso uso de memoria (MB) limitación Analysis  
 Este es el uso de memoria del proceso actual (MB). Limitación de memoria del proceso de BizTalk puede producirse si el lote que se va a publicar tiene requisitos de memoria elevados o si hay demasiados subprocesos se están procesando mensajes.  Si parece que el sistema sobrelimitado, considere la posibilidad de aumentar el valor asociado con el umbral de uso de memoria de proceso para el host y compruebe que la instancia de host no genera un error de "memoria insuficiente".  Si el servidor BizTalk server se ejecuta con regularidad fuera de la memoria virtual, a continuación, considere la posibilidad de BizTalk Server de 64 bits. Cada proceso en los servidores de 64 bits puede direccionar hasta 4 TB de memoria virtual frente a de los 2 GB 32 bits. En general, BizTalk de 64 bits y 64 bits de SQL Server se recomienda. Consulte la referencia de "compatibilidad con BizTalk Server de 64 bits" para obtener más información. Este análisis comprueba si el uso de memoria de proceso es mayor que 80 por ciento de su respectivo umbral de limitación del mismo nombre. De forma predeterminada, el uso de memoria de proceso de BizTalk configuración de limitación es el 25 por ciento de la memoria virtual disponible para el proceso.
