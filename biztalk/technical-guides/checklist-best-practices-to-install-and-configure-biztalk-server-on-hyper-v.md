# <a name="checklist-best-practices-for-installing-and-configuring-biztalk-server-on-hyper-v"></a>Lista de comprobación: Procedimientos recomendados para la instalación y configuración de BizTalk Server en Hyper-V
Las siguientes secciones son un resumen de la instalación y los requisitos de configuración se describen en el [implementación de BizTalk Server en Hyper-V](../technical-guides/deploying-biztalk-server-on-hyper-v.md) sección de esta guía. Deben utilizarse como una referencia rápida al instalar, configurar e implementar [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] en un entorno de Hyper-V. Se proporcionan vínculos a las secciones correspondientes para obtener más información.  

## <a name="before-installing-hyper-v"></a>Antes de instalar Hyper-V...  

|Paso|Referencia|  
|----------|---------------|  
|Hyper-V es un rol de servidor disponible para las ediciones de 64 bits de Windows Server.|Consulte [lista de comprobación: procedimientos recomendados para la instalación y configuración de BizTalk Server en Hyper-V](../technical-guides/checklist-best-practices-to-install-and-configure-biztalk-server-on-hyper-v.md)|  
|Asegúrese de que su procesador admite la virtualización asistida por hardware y prevención de ejecución de datos (DEP) y que estas características están habilitadas. Esto requiere un procesador que es compatible con Intel Virtualization Technology (Intel VT) o AMD Virtualization (AMD-V).|Consulte [instalar el rol de Hyper-V](https://docs.microsoft.com/windows-server/virtualization/hyper-v/get-started/install-the-hyper-v-role-on-windows-server).|  
|Usar Windows Server Core Edition para la partición raíz. Esto minimizará la sobrecarga del servidor y mejorar el rendimiento de Hyper-V.|Consulte [instalación de Server Core](https://docs.microsoft.com/windows-server/get-started/getting-started-with-server-core).|  
|Ejecutar solo el rol de servidor de Hyper-V en la partición raíz.|Consulte [servidores Hyper-V de optimización del rendimiento](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/):<br />**Rol de servidor dedicado**<br />La partición raíz debe estar dedicada a la función de servidor de virtualización. Otros roles de servidor pueden afectar negativamente al rendimiento del servidor de virtualización, especialmente si se consumen mucho ancho de banda de CPU, memoria o E/S. Minimizar los roles de servidor en la partición raíz tiene ventajas adicionales como la reducción de la superficie de ataque y la frecuencia de las actualizaciones. Los administradores del sistema deben considerar detenidamente qué software está instalado en la partición raíz porque algunos programas de software pueden afectar negativamente al rendimiento general del servidor de virtualización.<br/><br/> Consulte [configuración de Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/configuration) para obtener instrucciones.|  

## <a name="when-creating-hyper-v-virtual-machines"></a>Al crear máquinas virtuales de Hyper-V...  

|Paso|Referencia|  
|----------|---------------|  
|Con un disco duro virtual de tamaño fijo (VHD) proporciona un rendimiento mejorado en comparación con tamaño dinámicamente discos duros virtuales para las unidades del sistema operativo.|Consulte [rendimiento de E/S de almacenamiento de Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/storage-io-performance) para obtener instrucciones: <br />**VHD de tamaño fijo**<br />En primer lugar se asigna espacio para el disco duro virtual cuando se crea el archivo VHD. Este tipo de disco duro virtual es menos apt fragmentar, lo que reduce el rendimiento de E/S cuando se divide una E/S única en múltiples entradas y salidas. Tiene la menor sobrecarga de CPU de los tres tipos de disco duro virtual porque las lecturas y escrituras no es necesario buscar la asignación del bloque.|  
|Usar discos de la unidad de disco duro virtual (VHD) de tamaño fijo para las actividades de E/S de disco alta y configurar los discos para volúmenes de datos mediante la controladora SCSI. Para cargas de trabajo de E/S de almacenamiento de alto rendimiento que abarcan varias unidades de datos, cada disco duro virtual debe asociarse a una controladora SCSI sintética independiente para mejorar el rendimiento general. Además, cada disco duro virtual debe almacenarse en discos físicos separados.|Consulte [rendimiento de E/S de almacenamiento de Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/storage-io-performance) para obtener instrucciones:<br />**Controlador SCSI sintético**<br />El controlador de almacenamiento sintéticos proporciona un rendimiento significativamente mejor en las operaciones de E/s de almacenamiento con la menor sobrecarga de CPU que el dispositivo emulado del IDE. Los servicios de integración de la máquina virtual incluyen el controlador habilitado para este dispositivo de almacenamiento y son necesarios para el sistema operativo invitado detectarlo. Se debe montar el disco del sistema operativo en el dispositivo IDE para el sistema operativo arrancar correctamente, pero los servicios de integración de la máquina virtual carguen un controlador de filtro que redistribuye el dispositivo IDE las operaciones de E/s al dispositivo de almacenamiento sintético.<br />Se recomienda montar las unidades de datos directamente a la controladora SCSI sintética porque esa configuración ha reducido la sobrecarga de CPU. También debe montar archivos de registro y el archivo de paginación del sistema operativo directamente a la controladora SCSI sintético si su velocidad de E/S esperada es alta.<br />Para cargas de trabajo de E/S de almacenamiento de alto rendimiento que abarcan varias unidades de datos, cada disco duro virtual debe asociarse a una controladora SCSI sintética independiente para mejorar el rendimiento general. Además, cada disco duro virtual debe almacenarse en discos físicos separados.|  
|Usar el controlador de SCSI para conectar discos VHD para las actividades de E/S altos, como para los archivos de registro y datos de SQL Server. **Nota:** no adjuntar un disco del sistema a una controladora SCSI. Un disco duro virtual que contenga un sistema operativo debe asociarse a una controladora IDE.|Aunque la controladora IDE de Hyper-V y la controladora SCSI ofrecen un rendimiento comparable, solo puede instalarse la controladora SCSI Si Servicios de integración de Hyper-V están instalados. Por lo tanto, el uso de la controladora SCSI que se va a adjuntar discos de acceso directo garantizará que están instalados los servicios de integración de Hyper-V que a su vez garantiza un rendimiento E/S óptimo de los discos.|  
|Utilice el adaptador de red en lugar del adaptador de red heredado cuando la configuración de redes para una máquina virtual. El adaptador de red heredado está diseñado para los sistemas operativos que no son compatibles con los componentes de integración.|**Adaptador de red sintético**<br />Las características de Hyper-V para un adaptador de red sintéticos que está diseñado específicamente para que las máquinas virtuales lograr significativamente reducida CPU sobrecarga en E/S de red cuando se compara con el adaptador de red emulado que se asemeje a hardware existente. El adaptador de red sintético se comunica entre las particiones raíz y secundarios a través de VMBus mediante el uso de memoria compartida para la transferencia de datos más eficaz. El adaptador de red emulado deberá quitarla mediante el cuadro de diálogo de configuración de máquina virtual y reemplazado por un adaptador de red sintético. El invitado requiere que los servicios de integración de la máquina virtual esté instalado.|  
|Asegúrese de que los servicios de integración están instalados en los sistemas operativos de invitado habilitada y comprobación que la versión más reciente de integration services está instalada. Para comprobar la versión más reciente de integration services, ejecute **Windows Update**.|Consulte [el rendimiento del procesador de Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/processor-performance) para obtener instrucciones:<br />**Invitados habilitados**<br />Se recomienda que utilice Windows Server como sistema operativo invitado. Las mejoras de reducir la sobrecarga de CPU de Windows que se ejecuta en una máquina virtual. Los servicios de integración proporcionan giraría adicional para la E/S. Dependiendo de la carga del servidor, puede ser adecuado hospedar una aplicación de servidor en un invitado de Windows Server para mejorar el rendimiento.|  
|Siempre que sea posible, configure una asignación 1-1 de procesadores virtuales y los procesadores lógicos disponibles.|Para obtener más información acerca de cómo configurar una asignación 1 a 1 de procesadores virtuales y los procesadores lógicos disponibles, consulte la sección "Optimizar el rendimiento de procesador" en [lista de comprobación: optimización del rendimiento en Hyper-V](~/technical-guides/checklist-optimizing-performance-on-hyper-v.md).|  
|Convertir o migrar máquinas virtuales que se ejecutan en VMWare ESX Server se ejecute en Hyper-V, Microsoft Virtual Server o Microsoft Virtual PC.|-Use System Center Virtual Machine Manager para convertir o migrar máquinas virtuales se ejecuten en Hyper-V. <br />-Si es necesario, el proceso de conversión de máquinas virtuales que se ejecutan en Microsoft Virtual PC o Virtual Server de Microsoft puede realizarse manualmente. Para obtener más información, consulte [Guía de migración de máquina Virtual: How To Migrate de Virtual Server a Hyper-V](http://go.microsoft.com/fwlink/?LinkID=137258).<br />-La herramienta de ejemplo **VMC2Hyper V** también puede usarse para migrar máquinas virtuales que se ejecutan en Microsoft Virtual PC o Microsoft Virtual Server a Hyper-V. Para obtener más información acerca de la herramienta de ejemplo VMC2Hyper-V, consulte [VMC a la herramienta de importación de Hyper-V disponibles](http://go.microsoft.com/fwlink/?LinkID=135683). |
