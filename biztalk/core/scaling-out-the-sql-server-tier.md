## <a name="when-to-scale-out-the-messagebox-database"></a><span data-ttu-id="78d0c-121">Cuándo es recomendable escalar de forma horizontal a una base de datos de cuadro de mensajes</span><span class="sxs-lookup"><span data-stu-id="78d0c-121">When to Scale-out the MessageBox database</span></span>  
  
-   <span data-ttu-id="78d0c-122">En la base de datos de cuadro de mensajes se produce el cuello de botella.</span><span class="sxs-lookup"><span data-stu-id="78d0c-122">The MessageBox database becomes the bottleneck.</span></span> <span data-ttu-id="78d0c-123">Estos cuellos de botella pueden ser:</span><span class="sxs-lookup"><span data-stu-id="78d0c-123">Those bottlenecks can be:</span></span>

    -   <span data-ttu-id="78d0c-126">**Contención de bloqueo** escenarios complejos formados por varias orquestaciones o instancias de host tienden a crear contenciones de bloqueo en la base de datos de cuadro de mensajes.</span><span class="sxs-lookup"><span data-stu-id="78d0c-126">**Lock Contention** Complex scenarios with multiple host instances or orchestrations tend to create lock contention on the MessageBox database.</span></span> <span data-ttu-id="78d0c-127">En este caso, la agregación de otra base de datos de cuadro de mensajes de publicación también debería contribuir a incrementar el rendimiento.</span><span class="sxs-lookup"><span data-stu-id="78d0c-127">Again, adding another publishing MessageBox database should help increase the throughput.</span></span>
