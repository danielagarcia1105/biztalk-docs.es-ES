 <span data-ttu-id="80eb4-112">El elemento de las orquestaciones que hace que la publicación y la suscripción parezcan acciones diferentes es el enlace.</span><span class="sxs-lookup"><span data-stu-id="80eb4-112">The concept in orchestrations that makes the publish and subscribe seem to act differently is binding.</span></span> <span data-ttu-id="80eb4-113">Los puertos de orquestación son puertos lógicos que describen una interacción.</span><span class="sxs-lookup"><span data-stu-id="80eb4-113">Orchestration ports are logical ports that describe an interaction.</span></span> <span data-ttu-id="80eb4-114">Debe enlazar estos puertos lógicos a un puerto físico para que se lleve a cabo la entrega de los mensajes, aunque este proceso de enlace no sea otra cosa que configurar suscripciones para el enrutamiento de mensajes.</span><span class="sxs-lookup"><span data-stu-id="80eb4-114">You must bind these logical ports to a physical port so that messages are delivered, but this binding process is nothing more than configuring subscriptions for message routing.</span></span>  
  
 <span data-ttu-id="80eb4-115">Existen cuatro opciones básicas a la hora de enlazar estos puertos:</span><span class="sxs-lookup"><span data-stu-id="80eb4-115">There are four basic options for binding these ports:</span></span>

## <a name="deploying-orchestrations"></a><span data-ttu-id="80eb4-120">Implementar orquestaciones</span><span class="sxs-lookup"><span data-stu-id="80eb4-120">Deploying Orchestrations</span></span>  
 <span data-ttu-id="80eb4-121">Cuando se especifica un enlace en tiempo de diseño, el puerto físico que coincide con los parámetros configurados en la orquestación se crea durante la implementación de la orquestación.</span><span class="sxs-lookup"><span data-stu-id="80eb4-121">When a binding is specified at design time, the physical port that matches the parameters configured in the orchestration is created when the orchestration is deployed.</span></span> <span data-ttu-id="80eb4-122">Cuando el enlace se configura durante la implementación, los puertos que cumplan los requisitos del puerto lógico se pueden enlazar al puerto de orquestación.</span><span class="sxs-lookup"><span data-stu-id="80eb4-122">When the binding is configured at deployment time, any port that matches the requirements of the logical port can be bound to the orchestration port.</span></span> <span data-ttu-id="80eb4-123">En el enlace dinámico, un puerto físico se crea simplemente de forma similar a la opción Especificar ahora, pero el puerto es un puerto de envío dinámico en el que no se ha configurado la información de dirección.</span><span class="sxs-lookup"><span data-stu-id="80eb4-123">For dynamic binding, a physical port is created just as with the Specify Now option, but the port is a dynamic send port that has no address information configured.</span></span>  
  
 <span data-ttu-id="80eb4-124">Un concepto que confunde a menudo es el hecho de que, aunque un puerto de envío de una orquestación se enlaza a un puerto de envío físico, éste no impide que el mensaje para el mensaje que se envía al puerto enlazado, los dos puertos de envío reciben el mensaje.</span><span class="sxs-lookup"><span data-stu-id="80eb4-125">That is, if another send port happens to have a subscription, through its filters, for the message being sent to the bound port, both send ports receive the message.</span></span> <span data-ttu-id="80eb4-126">El enlace tan solo crea la suscripción de forma que el mensaje enviado desde la orquestación cumpla siempre los criterios del puerto de envío enlazado.</span><span class="sxs-lookup"><span data-stu-id="80eb4-126">Binding simply orquestación enlazado a un puerto de recepción crea la suscripción adecuada basado en el identificador de mensaje recepción y tipo de puerto.</span><span class="sxs-lookup"><span data-ttu-id="80eb4-128">Las suscripciones garantizan que los mensajes que entran y salen de la orquestación se entregan a los puertos enlazados, pero los mensajes aún vaya a través de la misma publicación y suscripción mecanismo que se ha descrito anteriormente.</span><span class="sxs-lookup"><span data-stu-id="80eb4-128">The subscriptions guarantee that the messages going in and out of the  <span data-ttu-id="80eb4-129">Probablemente, la opción de enlace de empleo inadecuado y menos frecuente, y que provoca mayores malentendidos, es la opción de Enlace directo.</span><span class="sxs-lookup"><span data-stu-id="80eb4-129">Probably the most misunderstood, and misused or underused binding option is the Direct binding option.</span></span> <span data-ttu-id="80eb4-130">El enlace directo permite a una orquestación publicar mensajes en la base de datos de cuadro de mensajes con propiedades de enrutamiento variables, tal y como ocurre con los mensajes publicados por ubicaciones de recepción.</span><span class="sxs-lookup"><span data-stu-id="80eb4-130">Direct binding allows an orchestration to publish messages to the MessageBox database with varying routing properties much like messages are published by receive locations.</span></span> <span data-ttu-id="80eb4-131">En la mensajería directa sencilla, el mensaje se publica en el cuadro de mensajes con sus propiedades promocionadas para el enrutamiento, como cualquier otro mensaje publicado que se recibe en BizTalk Server.</span><span class="sxs-lookup"><span data-stu-id="80eb4-131">In simple direct messaging, the message is published to the MessageBox with its promoted properties to be routed like any other published message received into BizTalk Server.</span></span> <span data-ttu-id="80eb4-132">Esto permite a los suscriptores recibir este mensaje, pero, al mismo tiempo, requiere que exista al menos un suscriptor, pues en caso contrario la orquestación recibirá un error en el enrutamiento.</span><span class="sxs-lookup"><span data-stu-id="80eb4-132">This enables any subscriber to receive this message, but requires that at least one subscriber exist or the orchestration will receive a routing failure error.</span></span>  
  
 <span data-ttu-id="80eb4-133">Otra opción para el enlace directo consiste en utilizar puertos que se autocorrelacionan.</span><span class="sxs-lookup"><span data-stu-id="80eb4-133">Another option for  <span data-ttu-id="80eb4-139">La última opción consiste en utilizar una orquestación de socio comercial, en la que en ambas orquestaciones el puerto esté configurado con el mismo tipo de puerto compartido y esté seleccionado el mismo puerto en la configuración del puerto.</span><span class="sxs-lookup"><span data-stu-id="80eb4-139">The final option is to use a partner orchestration in which, in both the calling orchestration and the called orchestration, the port is configured using the same shared port type and in the port configuration, the same port is selected.</span></span> <span data-ttu-id="80eb4-140">Por ejemplo, en Orch1 y en Orch2 se selecciona Orch2.MyDirectPort.</span><span class="sxs-lookup"><span data-stu-id="80eb4-140">For example, in both Orch1 and Orch2, Orch2.MyDirectPort is selected.</span></span> <span data-ttu-id="80eb4-141">Este tipo de enlace configura una suscripción para la orquestación de recepción, en función del tipo de orquestación de envío, el nombre de puerto y el nombre de operación.</span><span class="sxs-lookup"><span data-stu-id="80eb4-141">This type of binding sets up a subscription for the receiving orchestration based on the sending orchestration type, the port name, and the operation name.</span></span> <span data-ttu-id="80eb4-142">De nuevo, esto garantiza que los mensajes se enruten a la instancia correcta.</span><span class="sxs-lookup"><span data-stu-id="80eb4-142">This again ensures that the messages get routed to the correct instance.</span></span>  
  
 <span data-ttu-id="80eb4-143">Todas las opciones de mensajería directa utilizan la publicación subyacente y el modelo de suscripción.</span><span class="sxs-lookup"><span data-stu-id="80eb4-143">All of the direct messaging options use the underlying publish and subscribe model.</span></span> <span data-ttu-id="80eb4-144">La diferencia entre ellas radica en las propiedades que se utilizan en la creación de suscripciones y en el enrutamiento, así como en los casos de uso que ayuden a resolver.</span><span class="sxs-lookup"><span data-stu-id="80eb4-144">The difference between these options is in the properties that are used for creating subscriptions and routing, and in the use cases they help solve.</span></span>  
  
 <span data-ttu-id="80eb4-145">Un problema habitual del empleo de puertos de enlace directo se produce cuando la orquestación publica un mensaje al que también está suscrita.</span><span class="sxs-lookup"><span data-stu-id="80eb4-145">One common problem encountered when using direct bound ports in orchestrations is that an orchestration may publish a message that it is also subscribed to.</span></span> <span data-ttu-id="80eb4-146">Por ejemplo, una orquestación se configura para que se active con un mensaje PurchaseOrder.</span><span class="sxs-lookup"><span data-stu-id="80eb4-146">For example, an orchestration is configured to be activated by a PurchaseOrder message.</span></span> <span data-ttu-id="80eb4-147">Esta orquestación utiliza un puerto directo para publicar el mensaje PurchaseOrder en el cuadro de mensajes.</span><span class="sxs-lookup"><span data-stu-id="80eb4-147">This orchestration uses a direct port to publish the PurchaseOrder message to the MessageBox.</span></span> <span data-ttu-id="80eb4-148">No obstante, además de recibir el mensaje como se esperaba, se inicia otra instancia de una orquestación, dado que también poseía una suscripción para mensajes PurchaseOrder.</span><span class="sxs-lookup"><span data-stu-id="80eb4-148">However, in addition to receiving the message as expected, another instance of an orchestration is started because it too had a subscription for PurchaseOrder messages.</span></span> <span data-ttu-id="80eb4-149">El procesamiento entra en un bucle infinito y el programador puede tardar cierto tiempo en averiguar lo que ocurre.</span><span class="sxs-lookup"><span data-stu-id="80eb4-149">The processing gets into an endless loop and it may take some time for a developer to figure out what has happened.</span></span>  
  
## <a name="correlation"></a><span data-ttu-id="80eb4-150">Correlation</span><span class="sxs-lookup"><span data-stu-id="80eb4-150">Correlation</span></span>  
 <span data-ttu-id="80eb4-151">*Correlación* en orquestaciones es el mecanismo para recibir los mensajes relacionados en la misma instancia de orquestación de ejecución.</span><span class="sxs-lookup"><span data-stu-id="80eb4-151">*Correlation* in orchestrations is the mechanism for receiving related messages into the same running orchestration instance.</span></span> <span data-ttu-id="80eb4-152">En el Diseñador de orquestaciones, un programador sigue estos pasos generales para utilizar una correlación:</span><span class="sxs-lookup"><span data-stu-id="80eb4-152">In the Orchestration Designer a developer follows these general steps to use a correlation:</span></span>  
  
-   <span data-ttu-id="80eb4-153">Define un tipo de correlación que incluye las propiedades promocionadas utilizadas para relacionar mensajes.</span><span class="sxs-lookup"><span data-stu-id="80eb4-153">Defines a correlation type that includes the promoted properties that are used to relate messages.</span></span>  
  
-   <span data-ttu-id="80eb4-154">Define un conjunto de correlaciones que representa una instancia del tipo de correlación que se acaba de definir.</span><span class="sxs-lookup"><span data-stu-id="80eb4-154">Defines a correlation set that is an instance of the correlation type just defined.</span></span>  
  
-   <span data-ttu-id="80eb4-155">En los puertos de envío y recepción, especifica si inician o siguen un conjunto de correlaciones determinado.</span><span class="sxs-lookup"><span data-stu-id="80eb4-155">For the send and receive ports, specifies whether they initiate or follow a given correlation set.</span></span>  
  
 <span data-ttu-id="80eb4-156">Las suscripciones de instancia entran en juego cuando se inicia un conjunto de correlaciones, pues esto ocurre cuando se crean suscripciones para todos los puertos que siguen este conjunto de correlaciones para recibir mensajes.</span><span class="sxs-lookup"><span data-stu-id="80eb4-156">Instance subscriptions come into play when a correlation set is initiated, as this is when subscriptions are created for all of those ports that follow this correlation set to receive messages.</span></span> <span data-ttu-id="80eb4-157">Puesto que el tipo de correlación define las propiedades que se van a utilizar en la correlación, el motor de orquestaciones puede extraer estas propiedades del mensaje que la acción de iniciación está enviando o recibiendo.</span><span class="sxs-lookup"><span data-stu-id="80eb4-157">Because the correlation type defines the properties to be used for correlation, the orchestration engine can extract these properties from the message being sent or received by the initiating action.</span></span> <span data-ttu-id="80eb4-158">Estos valores se utilizan para definir suscripciones para el resto de las acciones restantes que siguen este conjunto de correlaciones.</span><span class="sxs-lookup"><span data-stu-id="80eb4-158">These values are then used to define subscriptions for all of the remaining actions which follow this correlation set.</span></span>  
  
 <span data-ttu-id="80eb4-159">Es importante que los mensajes que se reciben en BizTalk Server y que pretenden utilizarse en una correlación tengan las propiedades promocionadas correctamente definidas y promocionadas en el contexto del mensaje.</span><span class="sxs-lookup"><span data-stu-id="80eb4-159">It is important that messages received into BizTalk Server and intended for use in a correlation have their promoted properties correctly defined and promoted to the message context.</span></span> <span data-ttu-id="80eb4-160">La mayoría de las propiedades se promocionan cuando un componente de desensamblador de una canalización extrae los valores al recibirse el mensaje en un primer momento.</span><span class="sxs-lookup"><span data-stu-id="80eb4-160">Most properties get promoted when a disassembler component in a pipeline extracts the values when the message is initially received.</span></span> <span data-ttu-id="80eb4-161">Por esta razón no existe posibilidad alguna de utilizar la canalización de recepción PassThrough para recibir mensajes que deben correlacionarse con una instancia de orquestación en ejecución.</span><span class="sxs-lookup"><span data-stu-id="80eb4-161">For this reason, it is not possible to use the PassThrough receive pipeline to receive messages that must be correlated to a running instance of an orchestration.</span></span> <span data-ttu-id="80eb4-162">Este problema surge cuando se utiliza el adaptador de recepción SOAP para recibir mensajes correlacionados, puesto que la canalización PassThrough es el valor predeterminado para la canalización de recepción al utilizar el Asistente para publicación de servicios Web.</span><span class="sxs-lookup"><span data-stu-id="80eb4-162">This issue arises when you use the SOAP receive adapter to receive correlated messages, because the PassThrough pipeline is the default value for the receive pipeline when using the Web Services Publishing Wizard.</span></span>
