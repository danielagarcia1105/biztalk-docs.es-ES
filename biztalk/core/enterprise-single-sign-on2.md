 <span data-ttu-id="b9c45-118">En este ejemplo, una orquestación procesa un mensaje enviado por alguna aplicación a BizTalk Server y lo envía a una aplicación afiliada que se ejecute en un gran sistema (mainframe) IBM.</span><span class="sxs-lookup"><span data-stu-id="b9c45-118">In this example, a message sent by some application to BizTalk Server is processed by an orchestration, then sent to an affiliate application running on an IBM mainframe.</span></span> <span data-ttu-id="b9c45-119">La misión del inicio de sesión único empresarial es garantizar que las credenciales correctas, es decir, el nombre de usuario y la contraseña correspondientes, se envían con el mensaje a la aplicación afiliada.</span><span class="sxs-lookup"><span data-stu-id="b9c45-119">The job of Enterprise Single Sign-On is to make sure that the correct credentials (that is., the right username and password) are sent with the message when it is passed to the affiliate application.</span></span>  
  
## <a name="message-processing-with-an-sso-ticket"></a><span data-ttu-id="b9c45-120">Procesamiento de mensajes con un vale de SSO</span><span class="sxs-lookup"><span data-stu-id="b9c45-120">Message Processing With an SSO Ticket</span></span>  
 <span data-ttu-id="b9c45-121">Como muestra el diagrama, cuando un adaptador de recepción obtiene un mensaje, el adaptador puede solicitar un vale de SSO del servidor de SSO A (paso 1).</span><span class="sxs-lookup"><span data-stu-id="b9c45-121">As the diagram shows, when a receive adapter gets a message, the adapter can request an SSO ticket from SSO server A (step 1).</span></span> <span data-ttu-id="b9c45-122">Este vale cifrado contiene la identidad de Windows del usuario que realizó la solicitud y un periodo de tiempo en espera.</span><span class="sxs-lookup"><span data-stu-id="b9c45-122">This encrypted ticket contains the Windows identity of the user that made the request and a timeout period.</span></span> <span data-ttu-id="b9c45-123">(No se debe confundir con un vale de Kerberos, no es lo mismo.)
