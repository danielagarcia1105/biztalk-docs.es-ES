## <a name="improving-performance-with-nested-scopes"></a><span data-ttu-id="9ab9b-150">Mejorar el rendimiento con ámbitos anidados</span><span class="sxs-lookup"><span data-stu-id="9ab9b-150">Improving Performance with Nested Scopes</span></span>  
 <span data-ttu-id="9ab9b-151">Una de las características más notorias el **OrderBroker** orquestación es el uso de ámbitos anidados.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-151">One of the noticeable things about the **OrderBroker** orchestration is its use of nested scopes.</span></span> <span data-ttu-id="9ab9b-152">Uno de los objetivos de los ámbitos anidados es mejorar el rendimiento a través de la limitación de los puntos de persistencia.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-152">The nested scopes are there, in part, to improve performance by limiting the persistence points.</span></span>  
  
 <span data-ttu-id="9ab9b-153">El motor de orquestaciones guarda periódicamente el estado de toda la orquestación en puntos de ejecución que se denominan puntos de persistencia.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-153">The orchestration engine periodically saves the state of the entire orchestration at execution points called persistence points.</span></span> <span data-ttu-id="9ab9b-154">El motor de orquestaciones trata automáticamente varias formas de orquestación, incluidos los **enviar** formas, como puntos de persistencia.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-154">The orchestration engine automatically treats several orchestration shapes, including **Send** shapes, as persistence points.</span></span> <span data-ttu-id="9ab9b-155">Para obtener una lista de puntos de persistencia y obtener más información acerca de ellos, consulte [persistencia y el motor de orquestaciones](../core/persistence-and-the-orchestration-engine.md).</span><span class="sxs-lookup"><span data-stu-id="9ab9b-155">For a list of persistence points and more information about them, see [Persistence and the Orchestration Engine](../core/persistence-and-the-orchestration-engine.md).</span></span>  
  
 <span data-ttu-id="9ab9b-156">Con cinco **enviar** formas, el **OrderBroker** orquestación debe tener cinco puntos de persistencia.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-156">With five **Send** shapes, the **OrderBroker** orchestration should have five persistence points.</span></span> <span data-ttu-id="9ab9b-157">Sin embargo, al agrupar **enviar** formas dentro de un ámbito de transacción atómica, el motor reconoce solo necesita un punto de persistencia para el ámbito.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-157">However, when you group **Send** shapes inside an atomic transaction scope, the engine recognizes it only needs one persistence point for the scope.</span></span> <span data-ttu-id="9ab9b-158">Dado que cuatro de las **enviar** formas **OrderBroker** orquestación no forman parte de controladores de excepciones y nada debe hacerse después del envío, pueden incluirse en un ámbito de transacción atómica.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-158">Because four of the **Send** shapes in **OrderBroker** orchestration are not part of exception handlers and nothing needs to be done after the send, they can go in an atomic transaction scope.</span></span> <span data-ttu-id="9ab9b-159">Esto reduce el número de puntos de persistencia.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-159">This reduces the number of persistence points.</span></span> <span data-ttu-id="9ab9b-160">Para obtener más información acerca de las transacciones atómicas, vea [transacciones atómicas](../core/atomic-transactions.md).</span><span class="sxs-lookup"><span data-stu-id="9ab9b-160">For more information about atomic transactions, see [Atomic Transactions](../core/atomic-transactions.md).</span></span>

> [!TIP]
>  <span data-ttu-id="9ab9b-163">Se puede mejorar el rendimiento si se reduce al mínimo el número de puntos de persistencia de una orquestación.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-163">You can improve performance by minimizing the number of persistence points in an orchestration.</span></span> <span data-ttu-id="9ab9b-164">Puede agrupar **enviar** formas en una transacción atómica para producir una persistencia única punto para todos los **enviar** formas.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-164">You can group **Send** shapes in an atomic transaction to produce a single persistence point for all of the **Send** shapes.</span></span> <span data-ttu-id="9ab9b-165">Si se finalizan a la vez los ámbitos de transacciones anidadas, se creará un único punto de persistencia para las transacciones.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-165">Ending nested transaction scopes at the same time produces a single persistence point for the transactions.</span></span>  
  
 <span data-ttu-id="9ab9b-166">Un ámbito de transacción atómica no puede tener un controlador de la excepción.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-166">An atomic transaction scope cannot have an exception handler.</span></span> <span data-ttu-id="9ab9b-167">Por ello, la orquestación anida el ámbito atómico dentro de una transacción de larga ejecución.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-167">Because of this, the orchestration nests the atomic scope inside a long running transaction.</span></span> <span data-ttu-id="9ab9b-168">Esta transacción externa puede tener un controlador de excepciones y es este controlador que procesa una excepción de la **enviar** formas.</span><span class="sxs-lookup"><span data-stu-id="9ab9b-168">This outer transaction can have an exception handler and it is this handler that processes an exception from the **Send** shapes.</span></span>
