## <a name="overview"></a><span data-ttu-id="8bd8c-103">Información general</span><span class="sxs-lookup"><span data-stu-id="8bd8c-103">Overview</span></span>
<span data-ttu-id="8bd8c-104">Puesto que BizTalk Server procesa cada vez más datos en el sistema, el tamaño de la base de datos de seguimiento de BizTalk (BizTalkDTADb) continúa aumentando.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-104">As BizTalk Server processes more and more data on your system, the BizTalk Tracking (BizTalkDTADb) database continues to grow in size.</span></span> <span data-ttu-id="8bd8c-105">Un crecimiento descontrolado reduce el rendimiento del sistema y puede dar lugar a errores en el Servicio de descodificación de datos de seguimiento (TDDS).</span><span class="sxs-lookup"><span data-stu-id="8bd8c-105">Unchecked growth decreases system performance and may generate errors in the Tracking Data Decode Service (TDDS).</span></span> <span data-ttu-id="8bd8c-106">Además de los datos de seguimiento general, se pueden acumular también los mensajes controlados en la base de datos de cuadro de mensajes, lo que daría lugar a un rendimiento pobre del disco.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-106">In addition to general tracking data, tracked messages can also accumulate in the MessageBox database, causing poor disk performance.</span></span>  
  
<span data-ttu-id="8bd8c-107">BizTalk Server automatiza ambos procesos mediante el trabajo DTA Purge and Archive.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-107">BizTalk Server automates both processes using the DTA Purge and Archive job.</span></span> <span data-ttu-id="8bd8c-108">Archivar y purgar los datos de la base de datos de seguimiento de BizTalk permite mantener el sistema en buen estado y tener los datos de seguimiento archivados para usarlos en el futuro.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-108">By archiving and purging data from the BizTalk Tracking database, you can maintain a healthy system, as well as keep your tracking data archived for future use.</span></span> <span data-ttu-id="8bd8c-109">Puesto que los archivos de la base de datos de seguimiento de BizTalk se acumulan con el tiempo y consumen espacio en disco, se recomienda moverlos con regularidad a un recurso de almacenamiento secundario.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-109">Because BizTalk Tracking database archives accumulate over time and consume disk space, it is a good idea to move the BizTalk Tracking database archives to secondary storage on a regular basis.</span></span>  
  
 <span data-ttu-id="8bd8c-110">Cuando se purgan datos de la base de datos de seguimiento de BizTalk, el trabajo DTA Purge and Archive purga distintos tipos de información de seguimiento, como, por ejemplo, datos de seguimiento del motor de reglas, información de eventos de orquestación, así como de instancia de servicio y de mensaje.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-110">When you purge data from the BizTalk Tracking database, the DTA Purge and Archive job purges different types of tracking information such as message and service instance information, orchestration event information, and rules engine tracking data.</span></span>  
  
 <span data-ttu-id="8bd8c-111">La antigüedad de un registro de datos de seguimiento se basa en la hora en la que se insertaron los datos en la base de datos de seguimiento de BizTalk.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-111">The age of a tracking data record is based on the time the tracking data was inserted into the BizTalk Tracking database.</span></span> <span data-ttu-id="8bd8c-112">El trabajo DTA Purge and Archive utiliza la marca de tiempo para comprobar continuamente si el registro es anterior a la ventana de actividad de datos.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-112">The DTA Purge and Archive job uses the time stamp to continuously verify whether the record is older than the live window of data.</span></span> <span data-ttu-id="8bd8c-113">Después de cada período de ventana de actividad, la base de datos de seguimiento de BizTalk se archiva y se crea un archivo de almacenamiento nuevo.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-113">After every live window period, the BizTalk Tracking database is archived and a new archive file is created.</span></span> <span data-ttu-id="8bd8c-114">Todos los datos de seguimiento completados que sean anteriores al período de ventana activa se purgan en cada intervalo de trabajo del Agente SQL Server especificado por la programación de trabajo.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-114">At each SQL Server Agent job interval specified by the job schedule, all completed tracking data older than the live window period is purged.</span></span>  
  
 <span data-ttu-id="8bd8c-115">BizTalk Server utiliza los conceptos de purga condicional y purga incondicional.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-115">BizTalk Server uses the concept of a soft purge and a hard purge.</span></span> <span data-ttu-id="8bd8c-116">La purga condicional se utiliza para purgar instancias completadas, mientras que la purga incondicional se utiliza únicamente para purgar instancias que no se han completado.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-116">The soft purge is used to purge completed instances, while the hard purge is only used to purge incomplete instances.</span></span>

 <span data-ttu-id="8bd8c-118">En el trabajo DTA Archive and Purge, la suma de los parámetros LiveHours y LiveDays es la ventana de actividad de los datos que desea mantener en el entorno de BizTalk Server.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-118">In the DTA Archive and Purge job, the sum of the LiveHours and LiveDays parameters is the live window of data you want to maintain in your BizTalk Server environment.</span></span> <span data-ttu-id="8bd8c-119">Se purgan todos los datos asociados con una instancia completada anterior a esta ventana de actividad de datos.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-119">All data associated with a completed instance older than this live window of data is purged.</span></span> <span data-ttu-id="8bd8c-120">De forma predeterminada, el trabajo DTA Archive and Purge no se encuentra habilitado.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-120">By default, the DTA Archive and Purge job is not enabled.</span></span> <span data-ttu-id="8bd8c-121">Primero, es preciso configurar el trabajo y habilitarlo.</span><span class="sxs-lookup"><span data-stu-id="8bd8c-121">You must first configure and then enable the job.</span></span>
