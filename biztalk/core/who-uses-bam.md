# <a name="who-uses-bam"></a>¿Quién usa BAM?
Hay dos tipos de usuarios que interactúan con las funciones de trabajador de información de BAM: los usuarios empresariales y analistas de negocio.  
  
 Los analistas de negocios usan el Complemento de BAM para Excel con el fin de definir los procesos empresariales que desean supervisar. También lo usan para definir la forma en que los usuarios empresariales ven los datos recopilados por BAM. Por ejemplo, es posible que un analista de negocios desee presentar datos sobre pedidos y haga las siguientes preguntas interesantes sobre ellos:  
  
- ¿Cuántos pedidos de compra ha recibido la empresa durante este período (año, mes, semana, día, hora, etc.)?  
  
- ¿Cuál es el valor actual de los pedidos de compra activos?  
  
- ¿Cuál es el tiempo medio necesario para que un pedido de compra pase por las fases del proceso de pedidos?  
  
- ¿Cuántos pedidos de compra hay en una determinada fase del proceso?  
  
- ¿Cuántos clientes han encargado un determinado producto?  
  
- ¿Cuál es la suma de los pedidos de un determinado producto?  
  
- ¿Varían las cantidades pedidas según la región geográfica?  
  
  El analista de negocios usa estas preguntas para definir actividades que reflejan el tipo de información que BAM debe obtener del proceso empresarial.  
  
  Los usuarios empresariales usan BAM para ver los datos definidos por el analista de negocios y recopilados por BAM. Un usuario empresarial puede usar BAM con estos fines:  
  
- Ver datos sobre una instancia de proceso individual. Por ejemplo, si un analista de negocios ha creado una actividad para supervisar un proceso de pedido, un usuario empresarial puede usar BAM para ver datos sobre un pedido de compra en particular.  
  
- Ver datos sobre instancias de actividades relacionadas. Por ejemplo, la factura o los envíos asociados a un determinado pedido de compra.  
  
- Ver datos agregados en tiempo real o archivados sobre un proceso. Los datos agregados se extienden a lo largo del tiempo. Por ejemplo, un usuario puede ver el número de pedidos de compra con un valor inferior a 500 USD recibidos el último período (semana, mes, año, etc.) o el valor medio de los pedidos de compra recibidos hoy.  
  
- Ver datos de progreso, lo que permite al usuario ver datos de las distintas fases de un proceso, tal como el tiempo medio que tarda un pedido de compra en pasar de la fase Recibido a la fase Realización.
