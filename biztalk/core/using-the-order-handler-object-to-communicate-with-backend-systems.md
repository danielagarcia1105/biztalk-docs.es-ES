# <a name="using-the-order-handler-object-to-communicate-with-backend-systems"></a><span data-ttu-id="a70f5-102">Utilizando el objeto Orderhandler para comunicarse con sistemas de back-end</span><span class="sxs-lookup"><span data-stu-id="a70f5-102">Using the Order Handler Object to Communicate with Backend Systems</span></span>
<span data-ttu-id="a70f5-103">La solución Administración de procesos empresariales puede comunicarse de distintas formas con el sistema de servidor de pedidos heredado, el sistema de suministro de cable que recibe los pedidos finales.</span><span class="sxs-lookup"><span data-stu-id="a70f5-103">There are many ways in which the business process management solution could communicate with the legacy back-end order system, the cable provisioning system that receives the final orders.</span></span> <span data-ttu-id="a70f5-104">La solución usa los recursos remotos de .NET que se encuentran en Microsoft [!INCLUDE[btsDotNetFramework](../includes/btsdotnetframework-md.md)] para comunicarse con el sistema de suministro.</span><span class="sxs-lookup"><span data-stu-id="a70f5-104">The solution uses the .NET remoting facilities found in Microsoft [!INCLUDE[btsDotNetFramework](../includes/btsdotnetframework-md.md)] to communicate with the provisioning system.</span></span>  
  
 <span data-ttu-id="a70f5-105">La solución usa una técnica habitual: utilizar una interfaz para definir el objeto de acceso al sistema de servidor.</span><span class="sxs-lookup"><span data-stu-id="a70f5-105">The solution uses a common technique of using an interface to define the access object to the back-end system.</span></span> <span data-ttu-id="a70f5-106">Al colocar la interfaz en un ensamblado independiente, el ensamblado de cliente puede tener acceso al objeto remoto sin tener que obtener acceso al ensamblado compilado.</span><span class="sxs-lookup"><span data-stu-id="a70f5-106">By putting the interface in a separate assembly the client assembly can have access to the remote object without having to have access to the compiled assembly.</span></span>  
  
 <span data-ttu-id="a70f5-107">El **IOrderHandler** interfaz define los métodos para comunicarse con el sistema de pedidos de back-end.</span><span class="sxs-lookup"><span data-stu-id="a70f5-107">The **IOrderHandler** interface defines the methods to communicate with the backend order system.</span></span> <span data-ttu-id="a70f5-108">Esta interfaz incluye métodos para analizar, activar, cancelar y realizar pedidos.</span><span class="sxs-lookup"><span data-stu-id="a70f5-108">The interface includes methods for analyzing, activating, canceling, and completing orders.</span></span> <span data-ttu-id="a70f5-109">También proporciona un método para identificar el tipo de servicio, un método necesario cuando se cancela un pedido.</span><span class="sxs-lookup"><span data-stu-id="a70f5-109">It also provides a method for identifying the service type, a method needed when an order is canceled.</span></span>  
  
 <span data-ttu-id="a70f5-110">El **CableOrder1**, **CableOrder2**, y las orquestaciones satélite utilizan el **OrderHandlerWrapper** objeto que implementa **IOrderHandler**.</span><span class="sxs-lookup"><span data-stu-id="a70f5-110">The **CableOrder1**, **CableOrder2**, and satellite orchestrations use the **OrderHandlerWrapper** object that implements **IOrderHandler**.</span></span> <span data-ttu-id="a70f5-111">El **OrderHandlerWrapper** objeto, a su vez, invoca una instancia remota de un **OrderHandler** objeto proporcionado por el **CableProvisioningSystemServer** ejecutable.</span><span class="sxs-lookup"><span data-stu-id="a70f5-111">The **OrderHandlerWrapper** object, in turn, invokes a remote instance of an **OrderHandler** object provided by the **CableProvisioningSystemServer** executable.</span></span> <span data-ttu-id="a70f5-112">El uso del objeto contenedor satisface el requisito empresarial de utilizar funciones remotas .NET para la comunicación con el sistema de servidor de pedidos y, al mismo tiempo, permite usar las características de reintento de los componentes de control de excepciones.</span><span class="sxs-lookup"><span data-stu-id="a70f5-112">Using the wrapper object meets the business requirement of using .NET remoting to communicate with the back-end order system while, at the same time, enabling use of the retry features of the exception handling components.</span></span>  
  
 <span data-ttu-id="a70f5-113">Dado que uno debe ser capaz de serializar los objetos al que hace referencia en una orquestación, el **OrderHandlerWrapper** también se puede serializar.</span><span class="sxs-lookup"><span data-stu-id="a70f5-113">Because one must be able to serialize every object referenced in an orchestration, the **OrderHandlerWrapper** can also be serialized.</span></span> <span data-ttu-id="a70f5-114">Mediante el **OrderHandlerWrapper** aísla el código de serialización de las orquestaciones.</span><span class="sxs-lookup"><span data-stu-id="a70f5-114">Using the **OrderHandlerWrapper** isolates the serialization code from the orchestrations.</span></span>  
  
 <span data-ttu-id="a70f5-115">Si examina el código, verá que el **OrderHandlerWrapper** objeto implementa explícitamente la **ISerializable** interfaz.</span><span class="sxs-lookup"><span data-stu-id="a70f5-115">If you look at the code, you'll se that the **OrderHandlerWrapper** object explicitly implements the **ISerializable** interface.</span></span> <span data-ttu-id="a70f5-116">El objeto debe controlar su propia serialización porque utiliza un constructor no predeterminado.</span><span class="sxs-lookup"><span data-stu-id="a70f5-116">The object must handle its own serialization because it uses a non-default constructor.</span></span>  
  
 <span data-ttu-id="a70f5-117">La utilización de características remotas .NET para la comunicación con el sistema de servidor es más eficaz que la mensajería.</span><span class="sxs-lookup"><span data-stu-id="a70f5-117">Using .NET remoting to communicate with the backend system is more efficient than using messaging.</span></span> <span data-ttu-id="a70f5-118">Además, en comparación con una solución pura de mensajería, permite enlazar de una forma más eficaz las orquestaciones con el sistema de servidor.</span><span class="sxs-lookup"><span data-stu-id="a70f5-118">On the other hand, it binds the orchestrations more tightly to the back-end system than a pure messaging solution might.</span></span> <span data-ttu-id="a70f5-119">Las características remotas .NET impiden asimismo que la solución aproveche las ventajas de las características integradas de peticiones de reintento de BizTalk Server.</span><span class="sxs-lookup"><span data-stu-id="a70f5-119">Using .NET remoting also prevents the solution from taking advantage of the built-in BizTalk Server features for retrying requests.</span></span>
