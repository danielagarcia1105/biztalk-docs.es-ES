# <a name="requirements-phase-recommendations"></a><span data-ttu-id="ae1fb-102">Recomendaciones para la fase de requisitos</span><span class="sxs-lookup"><span data-stu-id="ae1fb-102">Requirements Phase Recommendations</span></span>
<span data-ttu-id="ae1fb-103">La principal entrega relacionada con la fase de requisitos es una especificación de requisitos, o una especificación funcional con los requisitos que incluyen objetivos de rendimiento.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-103">The primary deliverable associated with the requirements phase is a requirements specification, or a functional specification that includes requirements including performance goals.</span></span> <span data-ttu-id="ae1fb-104">Al determinar estos objetivos, es muy importante que participen los usuarios finales y los propietarios empresariales del sistema, a fin de garantizar la obtención de un perfil preciso del rendimiento.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-104">It is very important to involve the end-users and business owners of the system when determining these goals to assure that an accurate profile for performance is derived.</span></span>  
  
## <a name="establish-performance-criteria"></a><span data-ttu-id="ae1fb-105">Establecer los criterios de rendimiento</span><span class="sxs-lookup"><span data-stu-id="ae1fb-105">Establish Performance Criteria</span></span>  
 <span data-ttu-id="ae1fb-106">Desde la perspectiva del rendimiento, la parte más importante de la especificación funcional que se crea durante esta fase es la definición de objetivos de rendimiento detallados para el proyecto y el establecimiento de los criterios de versión para ese rendimiento.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-106">From a performance perspective, the most important part of the functional specification that is created during this phase is the definition of detailed performance goals for the project and the establishment of performance release criteria.</span></span> <span data-ttu-id="ae1fb-107">Existen tres componentes vitales para definir los criterios de rendimiento:</span><span class="sxs-lookup"><span data-stu-id="ae1fb-107">There are three critical components to defining performance criteria:</span></span>

- <span data-ttu-id="ae1fb-109">Un requisito de rendimiento asociado con la función de rendimiento.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-109">A performance requirement associated with the performance function.</span></span>  
  
- <span data-ttu-id="ae1fb-110">Una distribución de tamaños y tipos de archivo.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-110">A distribution of file sizes and types.</span></span>  
  
  <span data-ttu-id="ae1fb-111">Estos criterios se abordan en [¿qué es rendimiento sostenible?](../core/what-is-sustainable-performance.md)</span><span class="sxs-lookup"><span data-stu-id="ae1fb-111">These criteria are discussed in [What Is Sustainable Performance?](../core/what-is-sustainable-performance.md)</span></span>  
  
  <span data-ttu-id="ae1fb-112">Los criterios de versión para el rendimiento de una aplicación se extraen a partir de los objetivos de rendimiento.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-112">You derive the performance release criteria for an application from the performance goals.</span></span> <span data-ttu-id="ae1fb-113">Estos criterios representan un comportamiento conseguible y medible que se puede demostrar con pruebas.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-113">These criteria embody an achievable and measurable behavior that you can prove via testing.</span></span> <span data-ttu-id="ae1fb-114">La aplicación no pasará a la fase de versión hasta que se cumplan todos los criterios de versión o, en caso de que no sea posible, se hayan identificado todas las excepciones a los criterios de versión.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-114">The application will not move into release unless and until all release criteria have been met or, if not achievable, identified as exceptions to the release criteria.</span></span>

  <span data-ttu-id="ae1fb-117">Además, los casos de las pruebas de rendimiento se basarán en cómo hay que medir los criterios de versión, de modo que el grado de detalle de estos últimos debe ser suficiente para evitar cualquier confusión.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-117">In addition, performance test cases will be based on how the release criteria are to be measured, so the criteria must be detailed enough to avoid confusion.</span></span> <span data-ttu-id="ae1fb-118">Por ejemplo, al establecer un rendimiento concreto que debe lograrse, se debe incluir:</span><span class="sxs-lookup"><span data-stu-id="ae1fb-118">For example, when stating a specific throughput is to be achieved, it should include:</span></span>  
  
- <span data-ttu-id="ae1fb-119">El hardware en el que se ejecutará, por ejemplo: el número y tipo de servidores, el tipo y la velocidad del disco, etc.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-119">The hardware on which it must run, for example, the number and type of servers, disk speed/type, etc.</span></span>

  <span data-ttu-id="ae1fb-122">Para juzgar si un criterio de versión está bien constituido, cualquier persona debe ser capaz de leerlo tal y como esté documentado y entender cómo crear un caso de prueba que demuestre si se cumple.</span><span class="sxs-lookup"><span data-stu-id="ae1fb-122">To judge how well formed a release criteria is, anyone should be able look at the release criteria as documented and understand how to build a test case to prove the criteria.</span></span>
