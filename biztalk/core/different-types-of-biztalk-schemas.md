[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]<span data-ttu-id="a8dcf-103"> admite los siguientes cuatro tipos de esquemas:</span><span class="sxs-lookup"><span data-stu-id="a8dcf-103"> supports the following four types of schemas:</span></span>  
  
- <span data-ttu-id="a8dcf-104">**Esquema XML**.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-104">**XML schema**.</span></span> <span data-ttu-id="a8dcf-105">un esquema XML define la estructura de una clase de mensajes de instancia XML.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-105">An XML schema defines the structure of a class of XML instance messages.</span></span> <span data-ttu-id="a8dcf-106">Puesto que este tipo de esquema usa el lenguaje de definición de esquemas XML (XSD) para definir la estructura de un mensaje de instancia XML, y éste es el fin previsto de XSD, dichos esquemas usan XSD de un modo sencillo.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-106">Because this type of schema uses XML Schema definition (XSD) language to define the structure of an XML instance message, and this is the intended purpose of XSD, such schemas use XSD in a straightforward way.</span></span>  
  
- <span data-ttu-id="a8dcf-107">**Esquema de archivo sin formato**.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-107">**Flat file schema**.</span></span> <span data-ttu-id="a8dcf-108">un esquema de archivo sin formato define la estructura de una clase de mensajes de instancia que usan un formato de archivo sin formato, ya sea delimitado, posicional o una combinación de los anteriores.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-108">A flat file schema defines the structure of a class of instance messages that use a flat file format, either delimited or positional or some combination thereof.</span></span> <span data-ttu-id="a8dcf-109">Dado que las capacidades semánticas nativas de XSD no dar cabida a todos los requisitos para definir la estructura de mensajes de instancia de archivo sin formato, como los distintos tipos de delimitadores que se pueden usar para diversos registros y campos del archivo sin formato: BizTalk Server utiliza las capacidades de anotación de XSD para almacenar esta información adicional dentro de un esquema XSD.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-109">Because the native semantic capabilities of XSD do not accommodate all of the requirements for defining the structure of flat file instance messages—such as the various types of delimiters that might be used for different records and fields within the flat file—BizTalk Server uses the annotation capabilities of XSD to store this extra information within an XSD schema.</span></span> <span data-ttu-id="a8dcf-110">BizTalk Server define un eficaz conjunto de etiquetas de anotación específicas útiles para almacenar toda la información adicional necesaria.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-110">BizTalk Server defines a rich set of specific annotation tags that can be used to store all of the required additional information.</span></span>  
  
- <span data-ttu-id="a8dcf-111">**Esquema de sobres**.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-111">**Envelope schema**.</span></span> <span data-ttu-id="a8dcf-112">un esquema de sobre es un tipo especial de esquema XML.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-112">An envelope schema is a special type of XML schema.</span></span> <span data-ttu-id="a8dcf-113">Los esquemas de sobres se utilizan para definir la estructura de los sobres XML, que permiten incluir uno o más documentos empresariales XML en un único mensaje de instancia XML.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-113">Envelope schemas are used to define the structure of XML envelopes, which are used to wrap one or more XML business documents into a single XML instance message.</span></span> <span data-ttu-id="a8dcf-114">Al definir un esquema XML para que sea un esquema de sobre, es necesario configurar un par de propiedades adicionales, en función de factores tales como si hay más de un registro raíz definido en el esquema de sobre.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-114">When you define an XML schema to be an envelope schema, a couple of additional property settings are required, depending on such factors as whether there are more than one root record defined in the envelope schema.</span></span>  
  
- <span data-ttu-id="a8dcf-115">**Esquema de propiedad**.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-115">**Property schema**.</span></span> <span data-ttu-id="a8dcf-116">los esquemas de propiedades se utilizan con uno de los dos mecanismos existentes en BizTalk Server para lo que se conoce como promoción de propiedades.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-116">A data-stu-id="a8dcf-117">Property promotion is the process of copying specific values from deep within an instance message to the message context.</span></span> <span data-ttu-id="a8dcf-118">En el contexto del mensaje, varios componentes de BizTalk Server pueden tener un acceso más sencillo a estos valores.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-118">From the message context, these values are more easily accessed by various BizTalk Server components.</span></span> <span data-ttu-id="a8dcf-119">Estos componentes utilizan los valores para realizar acciones como el enrutamiento de mensajes.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-119">These components use the values to perform actions such as message routing.</span></span> <span data-ttu-id="a8dcf-120">Justo antes de que el mensaje de instancia se envíe al destino, los valores de las propiedades promocionadas también se pueden copiar en la otra dirección, desde el de más fácil acceso contexto del mensaje a los niveles profundos del mensaje de instancia.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-120">Promoted property values can also be copied in the data-ttu-id="a8dcf-121">Un esquema de propiedades es una versión simple de un esquema de BizTalk que desempeña un rol en el proceso de copiar las propiedades promocionadas entre el mensaje de instancia y el contexto del mensaje.</span><span class="sxs-lookup"><span data-stu-id="a8dcf-121">A property schema is a simple version of a BizTalk schema that plays a role in the process of copying
