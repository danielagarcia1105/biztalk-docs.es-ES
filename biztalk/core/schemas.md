## <a name="schema-deployment"></a><span data-ttu-id="19b4a-131">Implementación de esquemas</span><span class="sxs-lookup"><span data-stu-id="19b4a-131">Schema Deployment</span></span>

 <span data-ttu-id="19b4a-135">Al implementar un esquema, la base de datos de administración almacena el contenido de éste.</span><span class="sxs-lookup"><span data-stu-id="19b4a-135">When you deploy a schema, the Management database stores the contents of the schema.</span></span> <span data-ttu-id="19b4a-136">Puede implementar varios esquemas con el mismo espacio de nombres de destino.</span><span class="sxs-lookup"><span data-stu-id="19b4a-136">You can deploy multiple schemas with the same target namespace.</span></span> <span data-ttu-id="19b4a-137">BizTalk Server señala de forma explícita al esquema en el Diseñador de canalizaciones que se utiliza en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="19b4a-137">BizTalk Server explicitly points to the schema in the Pipeline Designer that is used at run time.</span></span> <span data-ttu-id="19b4a-138">Si usa canalizaciones predeterminadas o no especifica un esquema en el Diseñador de canalizaciones y se implementan varias versiones del mismo esquema, BizTalk Server determina el esquema que se debe usar.</span><span class="sxs-lookup"><span data-stu-id="19b4a-138">If you use default pipelines, or if you do not specify a schema in the Pipeline Designer, and if multiple versions of the same schema have been deployed, BizTalk Server determines the schema to use.</span></span> <span data-ttu-id="19b4a-139">En este caso, será el esquema asociado a la versión más reciente, con el mayor número de versión: del ensamblado implementado con ese esquema.</span><span class="sxs-lookup"><span data-stu-id="19b4a-139">In this case, it will be the schema associated with the most recent version—with the highest version number—of the assembly deployed using that schema.</span></span>
