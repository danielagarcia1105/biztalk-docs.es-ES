# <a name="adding-roles-and-departments"></a>Agregar funciones y departamentos
Después de crear [!INCLUDE[btaA4SWIFT2.3abbrevnonumber](../../includes/btaa4swift2-3abbrevnonumber-md.md)] los usuarios que se cree, reparar, compruebe o aprobar mensajes, debe crear uno o varios departamentos con roles para establecer el flujo de trabajo de reparación/create.  
  
 Cuando se envía un mensaje para reparar, [!INCLUDE[btaA4SWIFT2.3abbrevnonumber](../../includes/btaa4swift2-3abbrevnonumber-md.md)] asigna un departamento para reparar el mensaje. [!INCLUDE[btaA4SWIFT2.3abbrevnonumber](../../includes/btaa4swift2-3abbrevnonumber-md.md)]usa las reglas de negocios para determinar qué departamento debe reparar los mensajes. Puede configurar las reglas de negocios para ejecutar flujos de trabajo de departamento según el tipo de mensaje o de otros datos. Para obtener más información sobre la configuración de los flujos de trabajo de departamento, consulte [crear departamentos y Roles para la reparación de mensajes y nuevo envío](../../adapters-and-accelerators/accelerator-swift/creating-departments-and-roles-for-message-repair-and-new-submission.md).  
  
 Consulte la documentación del cliente de Web de perfil ubicado en \<unidad >: \Program Acelerador de BizTalk para SWIFT\SDK\Samples\CustomMRSRModel para establecer las propiedades de A4SWIFT.
