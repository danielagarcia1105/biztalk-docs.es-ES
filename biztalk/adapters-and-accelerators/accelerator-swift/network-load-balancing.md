# <a name="network-load-balancing"></a>Equilibrio de carga de red
Una implementación distribuida de A4SWIFT puede contener equipos con BizTalk Server para mensajería, orquestación, servidores de SharePoint para el sitio MRSR de host o bases de datos de SQL Server. Si sus necesidades de negocio y el perfil de uso son lo suficientemente grandes, debe configurar la red equilibrio de carga (NLB) para dos o más HTTP de BizTalk front-end (MRSR) servidores de sitio o dos o más servidores de mensajería de BizTalk. Para obtener un ejemplo de varios servidores que necesita el equilibrio de carga de red, consulte el diagrama "Ejemplo de una implementación completa de A4SWIFT" en [diagrama físico](../../adapters-and-accelerators/accelerator-swift/physical-diagram.md).  
  
 Para obtener más información sobre el equilibrio de carga de red, consulte la Guía de implementación de BizTalk Server, y [paso 2: configurar NLB en los servidores](../../adapters-and-accelerators/accelerator-swift/step-2-configuring-nlb-on-the-servers.md).
