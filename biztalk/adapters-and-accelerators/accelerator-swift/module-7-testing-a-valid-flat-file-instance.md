# <a name="module-7-testing-a-valid-flat-file-instance"></a>Módulo 7: Probar una instancia de archivo sin formato válido
En este módulo, que envía un ejemplo válido MT103 creados en las prácticas anteriores de puertos de recepción de archivo sin formato en el archivo. Esta tarea comprueba las canalizaciones de recepción que creó en los laboratorios de anteriores. [!INCLUDE[btsCoName](../../includes/btsconame-md.md)][!INCLUDE[A4SWIFT_CurrentVersion_FirstRef](../../includes/a4swift-currentversion-firstref-md.md)]Escribe la salida en formato XML en la carpeta de salida en el puerto de envío seleccionado en la lección anterior, [lección 2: agregar un puerto de envío XML](../../adapters-and-accelerators/accelerator-swift/lesson-2-adding-an-xml-send-port.md).  
  
 Iniciar el archivo de adaptador de recepción mediante la copia de un archivo con formato sin formato SWIFT a la carpeta entrante. Esta acción da como resultado [!INCLUDE[btaA4SWIFT2.3abbrevnonumber](../../includes/btaa4swift2-3abbrevnonumber-md.md)] enrutamiento de un archivo XML de SWIFT válido a la carpeta de salida.  
  
 En esta tarea, también se envía un mensaje de MT103 que no es válido. Utilice el evento para resolver el error.
