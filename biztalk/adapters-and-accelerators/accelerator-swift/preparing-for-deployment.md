# <a name="preparing-for-deployment"></a>Preparación para la implementación
Esta sección proporciona información acerca de la fase de planeación y preparación de la implementación. Antes de implementar una implementación, realizar los preparativos siguientes:  
  
1. Leer y comprender los problemas conocidos y preparar el equipo:  
  
   -   Crear una lista de comprobación del hardware y software necesarios para la implementación y una hoja de diagrama y la implementación de red que reflejan la variación de la implementación (basada en el diagrama de red y de la arquitectura de implementación que ha seleccionado).  
  
   -   Adquirir el hardware y software en la lista de comprobación. Esta adquisición incluye la descarga de todos los service packs, revisiones y actualizaciones de software que son necesarias para la implementación.  
  
        Descargar estos componentes de software con un equipo con acceso a Internet y, a continuación, copie estos componentes de software en un CD-ROM para su distribución sea más fácil.  
  
2. Separar la implementación en fases diferentes e identificar las tareas asociadas con cada fase. Esta estrategia simplifica el proceso de implementación mediante la organización de las tareas según una fase de implementación específicos. Por ejemplo, la implementación recomendada incluye las siguientes fases de implementación de servidor:  
  
3. Configurar el controlador de dominio.  
  
4. Configuración de las bases de datos (incluido el tiempo de ejecución [!INCLUDE[btsSQLServerNoVersion](../../includes/btssqlservernoversion-md.md)] tiempo de diseño y el clúster de servidor de base de datos).  
  
5. Configuración de los servidores de BizTalk.
