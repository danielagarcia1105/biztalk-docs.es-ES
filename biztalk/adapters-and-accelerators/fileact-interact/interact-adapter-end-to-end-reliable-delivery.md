# <a name="interact-adapter-end-to-end-reliable-delivery"></a><span data-ttu-id="6ac4f-102">Interactuar entrega confiable de adaptador-to-End</span><span class="sxs-lookup"><span data-stu-id="6ac4f-102">InterAct Adapter End-to-End Reliable Delivery</span></span>
<span data-ttu-id="6ac4f-103">Al enviar mensajes o archivos a un destinatario, se recomienda que se asegure de que se entrega el mensaje o el archivo y que la empresa transacciones contenidas en estas se ejecutan no veces más que prever.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-103">When sending messages or files to a recipient, we recommend that you ensure that the message or file is delivered, and that the business transaction(s) contained in these are executed no more times than anticipated.</span></span>  
  
 <span data-ttu-id="6ac4f-104">Cuando ambas entidades que se comunican entre sí pueden usar un almacenamiento persistente (por ejemplo, proporcionada por un middleware orientado a mensajes persistente y una aplicación de interfaz con él), resulta fácil de implementar una entrega confiable si el modo de comunicarse el está normalizado percibido estado del mensaje.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-104">When both entities communicating with each other can use a persistent storage (for example, provided by a persistent message-oriented middleware and an interface application using it), it is easy to implement a reliable delivery if the way to communicate the perceived status of the message is standardized.</span></span>  
  
 <span data-ttu-id="6ac4f-105">En la siguiente ilustración muestra un ejemplo de la estructura de la E2EControl.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-105">The following figure shows an example of the structure of the E2EControl.</span></span>

 <span data-ttu-id="6ac4f-110">La manera de construir este identificador se deja para el implementador, pero normalmente se basa en una llamada del sistema, como uuidgen() o puede ser el resultado de calcular un SHA-1 en el que se envíe la solicitud (con un Sw:MsgId con prefijo y reemplace; para ello base64 codificado s SHA-1 String).</span><span class="sxs-lookup"><span data-stu-id="6ac4f-110">The way this identifier is constructed is left to the implementer, but typically it is based on a system call such as uuidgen(), or it may be the result of computing a SHA-1 on the request to be sent (with a prefixed Sw:MsgId and then replace it by the base64 encoded SHA-1 string).</span></span> <span data-ttu-id="6ac4f-111">El requisito principal es que es único globalmente (con una probabilidad muy alta).</span><span class="sxs-lookup"><span data-stu-id="6ac4f-111">The main requirement is that it is globally unique (with a very high probability).</span></span>  
  
 <span data-ttu-id="6ac4f-112">El Sw:CreationTime es el momento de creación de la solicitud original.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-112">The Sw:CreationTime is the time of creation of the original request.</span></span> <span data-ttu-id="6ac4f-113">Es un parámetro opcional, pero resulta útil limitar la eventual busca los intentos de comunicación anterior de este mensaje.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-113">It is an optional parameter, but it is useful to limit eventual searches for previous communication attempts of this message.</span></span>  
  
 <span data-ttu-id="6ac4f-114">El elemento Sw:PDIndication está presente para indicar que se trata de un segundo o más intento de transmitir el mensaje.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-114">The element Sw:PDIndication is present to indicate that this is a second or further attempt to transmit the message.</span></span> <span data-ttu-id="6ac4f-115">La aplicación receptora que conoce el E2EControl, a continuación, puede usar el Sw:MsgId para buscar de nuevo si se ha recibido el mensaje o no.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-115">The receiving application that is aware of the E2EControl can then use the Sw:MsgId to find back whether the message has been received or not.</span></span> <span data-ttu-id="6ac4f-116">El Sw:EmissionList opcional contiene la hora de los intentos anteriores.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-116">The optional Sw:EmissionList contains the time of the previous attempts.</span></span> <span data-ttu-id="6ac4f-117">Este tiempo es la hora local del remitente (en hora universal) ya que se obtuvo el remitente cuando se usa la función Sw:GetDateTime.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-117">This time is the local time of the sender (in universal time) as got by the Sender when using the Sw:GetDateTime function.</span></span> <span data-ttu-id="6ac4f-118">Nuevo Esto puede resultar útil para limitar las búsquedas.</span><span class="sxs-lookup"><span data-stu-id="6ac4f-118">Again this could be useful to limit searches.</span></span>
