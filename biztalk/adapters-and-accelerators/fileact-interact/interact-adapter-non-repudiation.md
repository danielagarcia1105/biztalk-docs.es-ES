# <a name="interact-adapter-non-repudiation"></a><span data-ttu-id="32236-102">Interactuar sin repudio de adaptador</span><span class="sxs-lookup"><span data-stu-id="32236-102">InterAct Adapter Non-Repudiation</span></span>
<span data-ttu-id="32236-103">Se obtiene compatibilidad sin repudio para un mensaje saliente de InterAct estableciendo la SwInt:NRIndicator como TRUE en la SwInt:RequestControl o SwInt:ResponseControl, según corresponda.</span><span class="sxs-lookup"><span data-stu-id="32236-103">Non-repudiation support for an outgoing InterAct message is obtained by setting the SwInt:NRIndicator to TRUE in the SwInt:RequestControl or SwInt:ResponseControl, as appropriate.</span></span> <span data-ttu-id="32236-104">Esto es necesario sólo si el servicio no selecciona sin repudio soporte de forma predeterminada, según el perfil de servicio.</span><span class="sxs-lookup"><span data-stu-id="32236-104">This is required only if the service does not select non-repudiation support by default, according to the Service Profile.</span></span>  
  
 <span data-ttu-id="32236-105">Sin repudio se basa en una firma válida creada por el SNL, justo antes de transmitir el mensaje.</span><span class="sxs-lookup"><span data-stu-id="32236-105">Non-repudiation is based on a valid signature created by the SNL, just before transmitting the message.</span></span> <span data-ttu-id="32236-106">Por lo tanto, para obtener soporte técnico sin repudio en los mensajes de solicitud salientes, es obligatorio para el elemento SwInt:RequestCrypto, dentro de la SwInt:RequestControl, que se establecerán TRUE en la SwInt:RequestControl.</span><span class="sxs-lookup"><span data-stu-id="32236-106">Therefore, in order to obtain non-repudiation support on outgoing request messages, it is mandatory for the SwInt:RequestCrypto element, contained within the SwInt:RequestControl, to be set TRUE in the SwInt:RequestControl.</span></span>  
  
 <span data-ttu-id="32236-107">Para los mensajes de respuesta, el requisito es equivalente, salvo que los elementos implicados son SwInt:ResponseCrypto, como dentro de la SwInt:ResponseControl; SwInt:ResponseCrypto se debe establecer en TRUE.</span><span class="sxs-lookup"><span data-stu-id="32236-107">For response messages, the requirement is equivalent, except that the elements involved are the SwInt:ResponseCrypto, as contained within the SwInt:ResponseControl; SwInt:ResponseCrypto must be set TRUE.</span></span>  
  
 <span data-ttu-id="32236-108">Para obtener soporte sin repudio en una solicitud, es necesario para la firma del mensaje cubrir la SwInt:RequestHeader, SwInt:RequestPayload y el SwInt:SwiftRequestRef de la SwInt:RequestDescriptor.</span><span class="sxs-lookup"><span data-stu-id="32236-108">For non-repudiation support on a request, it is necessary for the message signature to cover the SwInt:RequestHeader, SwInt:RequestPayload, and the SwInt:SwiftRequestRef of the SwInt:RequestDescriptor.</span></span> <span data-ttu-id="32236-109">Vínculo de SWIFTNet genera automáticamente el SwInt:SwiftRequestRef.</span><span class="sxs-lookup"><span data-stu-id="32236-109">The SwInt:SwiftRequestRef is automatically generated by SWIFTNet Link.</span></span> <span data-ttu-id="32236-110">En relación con la generación de la SwInt:SwiftRequestRef, la SNL ajusta también automáticamente el valor de SwSec:MemberRef dentro de la SwSec:CryptoControl para generar la firma del mensaje requerido.</span><span class="sxs-lookup"><span data-stu-id="32236-110">In connection with generating the SwInt:SwiftRequestRef, the SNL also automatically adjusts the SwSec:MemberRef value within the SwSec:CryptoControl for generating the required message signature.</span></span> <span data-ttu-id="32236-111">De forma similar, para no repudiatin soporte técnico en una respuesta, es necesario para la firma del mensaje cubrir la SwInt:ResponseHeader, SwInt:ResponsePayload y el SwInt:SwiftResponseRef de la SwInt:ResponseDescriptor.</span><span class="sxs-lookup"><span data-stu-id="32236-111">Similarly, for non-repudiatin support on a Response, it is necessary for the message signature to cover the SwInt:ResponseHeader, SwInt:ResponsePayload, and the SwInt:SwiftResponseRef of the SwInt:ResponseDescriptor.</span></span> <span data-ttu-id="32236-112">Vínculo de SWIFTNet genera automáticamente el SwInt:SwiftResponseRef.</span><span class="sxs-lookup"><span data-stu-id="32236-112">The SwInt:SwiftResponseRef is automatically generated by SWIFTNet Link.</span></span> <span data-ttu-id="32236-113">En relación con la generación de la SwInt:SwiftResponseRef, la SNL ajusta también automáticamente el valor de SwSec:MemberRef dentro de la SwSec:CryptoControl para generar la firma del mensaje requerido.</span><span class="sxs-lookup"><span data-stu-id="32236-113">In connection with generating the SwInt:SwiftResponseRef, the SNL also automatically adjusts the SwSec:MemberRef value within the SwSec:CryptoControl for generating the required message signature.</span></span>  
  
 <span data-ttu-id="32236-114">Si el perfil de servicio de negocio selecciona sin repudio de forma predeterminada, la firma del mensaje es necesario sigue siendo necesaria para seleccionarse (como se describe, en la SwInt:RequestControl o la SwInt:ResponseControl) y debe seleccionarse antes de que el mensaje abandona SNL.</span><span class="sxs-lookup"><span data-stu-id="32236-114">If the business Service Profile selects non-repudiation by default, the necessary message signature is still required to be selected (as just described, in the SwInt:RequestControl or the SwInt:ResponseControl), and must be selected before the message leaves the SNL.</span></span>  
  
 <span data-ttu-id="32236-115">Si la selección de características según el perfil de servicio de negocio invoca soporte sin rechazo en un mensaje y, si no se encuentra la firma necesaria con el mensaje, a continuación, se rechazará el mensaje mediante el conmutador.</span><span class="sxs-lookup"><span data-stu-id="32236-115">If Feature Selection according to the business Service Profile invokes Non-repudiation Support on a message, and if the necessary signature is not found with the message, then the message will be rejected by the Switch.</span></span> <span data-ttu-id="32236-116">Se devolverá un mensaje de excepción de estado al remitente del mensaje.</span><span class="sxs-lookup"><span data-stu-id="32236-116">A status exception message will be returned to the Sender of the message.</span></span>  
  
 <span data-ttu-id="32236-117">Cifrado de la carga no es coherente con el soporte técnico sin repudio.</span><span class="sxs-lookup"><span data-stu-id="32236-117">Payload encryption is not consistent with Non-repudiation Support.</span></span> <span data-ttu-id="32236-118">Si se selecciona soporte sin rechazo de un mensaje y la carga se cifra en su totalidad o en parte, se rechazará el mensaje por el vínculo de SWIFTNet.</span><span class="sxs-lookup"><span data-stu-id="32236-118">If Non-repudiation Support is selected for a message and the payload is encrypted in whole or in part, then the message will be rejected by SWIFTNet Link.</span></span>  
  
 <span data-ttu-id="32236-119">Tenga en cuenta que si el servicio no tiene la característica sin repudio, cualquier solicitud o respuesta que indica sin repudio en el control, se rechazarán.</span><span class="sxs-lookup"><span data-stu-id="32236-119">Note that if the service does not have the non-repudiation feature, any request or response indicating non repudiation in the control, will be rejected.</span></span>
