# <a name="key-features-in-biztalk-adapter-for-oracle-database"></a>Características clave en el adaptador de BizTalk para Oracle Database
Esta sección enumeran las características nuevas y en desuso en [!INCLUDE[adapteroracle](../../includes/adapteroracle-md.md)].  

  
## <a name="technology-related-features"></a>Características relacionadas con la tecnología  
  
|                   Característica                    |                                                                                                                                                                                                                                                                                                                                                                                                     Comentario                                                                                                                                                                                                                                                                                                                                                                                                     |
|----------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Nueva manera de conectarse a la base de datos de Oracle | Aparte de conectar a la base de datos de Oracle utilizando el nombre de servicio de red en el archivo tnsnames.ora (como se muestra en la versión anterior del adaptador), el adaptador de los clientes ahora también pueden conectarse a la base de datos de Oracle directamente mediante la especificación de los parámetros de conexión y, por tanto, lo que elimina la necesidad de usar un nombre de servicio de red o el archivo tnsnames.ora. No se requiere el archivo tnsnames.ora para conectarse a la base de datos de Oracle le ahorra evita la incomodidad de actualización manual de los parámetros de conexión (nombre de servicio de red) en el archivo tnsnames.ora en todos los equipos cliente al agregar o actualizar los servidores de Oracle en su entorno. Para obtener más información, consulte [crear una conexión a la base de datos de Oracle](../../adapters-and-accelerators/adapter-oracle-database/create-a-connection-to-the-oracle-database.md). |
|      Compatibilidad con la autenticación de Windows      |                                                                                                Los clientes del adaptador pueden usar la autenticación de Windows para conectarse a la base de datos de Oracle. Autenticación de Windows le permite determinar la identidad del usuario según las credenciales de inicio de sesión de Windows y, por tanto, le ayuda a aprovechar la seguridad del entorno de Windows integrada. Para obtener más información sobre la autenticación de Windows en el [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)], consulte [conectar con la autenticación de Windows de base de datos de Oracle utilizando](../../adapters-and-accelerators/adapter-oracle-database/connect-to-the-oracle-database-using-windows-authentication.md).                                                                                                |

## <a name="changes-to-note"></a>Tenga en cuenta los cambios

### <a name="general"></a>General  
* Para los parámetros de tipo IN OUT REF CURSOR

    -   Si se ha producido ningún cambio en el valor REF CURSOR dentro del procedimiento almacenado, el valor de la salida es el mismo que el valor de la entrada de REF CURSOR.  
  
    -   Los datos de entrada y salidos de REF CURSOR son del mismo tipo.  
  
* Un comportamiento incorrecto del atributo "nil": para todos los tipos de datos simples, si establece el valor del atributo nil en "true" y un valor para el campo o parámetro está presente, a continuación, el adaptador de base de datos de Oracle pasa incorrectamente el valor especificado en lugar de NULL. Como alternativa, si desea pasar el valor NULL para un campo o parámetro, debe asegurarse de que se especifica ningún valor para el campo o parámetro. Por ejemplo, para pasar el valor NULL para un campo denominado "name":  
  
  ```  
  <name xsi:nil="true"/>  
  ```  
* No se truncan Real, Float y tipos de datos largos y ceros adicionales (0) al final del valor del conjunto de resultados de la operación de selección. Además, el conjunto de resultados de la operación de selección siempre devuelve un valor con una precisión de 8 para el Real, Float y tipos de datos largos.  
  
* Control de los datos para los tipos de registros: el valor pasado para estos nodos dependen del valor de la **SkipNilNodes** enlaza la propiedad. Para obtener más información acerca de esta propiedad de enlace, consulte [configurar las propiedades de enlace para la base de datos de Oracle](../../adapters-and-accelerators/adapter-oracle-database/configure-the-binding-properties-for-oracle-database.md). 
  
* Las operaciones salientes: no se envía ningún valor para los parámetros que no tienen un valor especificado en el archivo XML de entrada. Si se especifica un valor predeterminado en el procedimiento almacenado, la base de datos de Oracle utiliza ese valor porque el adaptador no envió ningún valor. Si debe enviarse un valor NULL, el usuario debe especificar un nodo NULL del archivo de entrada XML estableciendo el valor del atributo "nil" en "true".  
  
* Es compatible con el tiempo de espera de comando.  
  
* La operación UpdateLOB debe realizarse como parte de una transacción. Para asegurarse de esto, el valor de la **UseAmbientTransaction** enlaza la propiedad debe establecerse en **True**.  
  
### <a name="biztalk-scenario"></a>Escenario de BizTalk  
  
- Las operaciones salientes: si el **UseAmbientTransaction** enlaza la propiedad es "True", las operaciones en la base de datos de Oracle y en BizTalk MessageBox base de datos se realizan dentro de la misma transacción distribuida. Para obtener más información acerca de las transacciones en el [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)], consulte [controlar las transacciones con el adaptador de base de datos de Oracle](../../adapters-and-accelerators/adapter-oracle-database/handle-transactions-with-the-oracle-database-adapter.md).  
  
- Operaciones entrantes: no se puede usar una respuesta de solicitud de puerto de recepción [!INCLUDE[btsBizTalkServerNoVersion](../../includes/btsbiztalkservernoversion-md.md)] para las operaciones de entrada mediante el [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)]. Unidireccional solo recibe se pueden usar los puertos.  
  
### <a name="other-scenarios"></a>Otros escenarios  
  
- Las operaciones de salida: el adaptador no inicia una transacción. Si el usuario desea varias filas para insertarse en la misma transacción, es responsabilidad del usuario para ejecutar la operación dentro de un ámbito de transacción de System.Transactions. El usuario también debe establecer el valor de la **UseAmbientTransaction** propiedad **True**. Para obtener más información acerca de las transacciones en el [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)], consulte [controlar las transacciones con el adaptador de base de datos de Oracle](../../adapters-and-accelerators/adapter-oracle-database/handle-transactions-with-the-oracle-database-adapter.md).  
  
- Las operaciones salientes: no pueden realizarse operaciones de Sll realizadas en el mismo objeto IRequestChannel/proxy en la misma conexión física a la base de datos de Oracle.  
  
- Modelo del canal de WCF: La [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)] no admite IReplyChannel al usar el modelo del canal de WCF. Sin embargo, puede usar IInputChannel para realizar operaciones de entrada. Además, con respecto a las transacciones, el adaptador se basa en la transacción del distribuidor WCF iniciado para ejecutar la instrucción de sondeo y registrar la instrucción de sondeo en la base de datos de Oracle. El nivel de aislamiento de transacción y el tiempo de espera de la transacción del distribuidor WCF iniciada pueden controlarse estableciendo los valores adecuados en el ServiceBehavior.
