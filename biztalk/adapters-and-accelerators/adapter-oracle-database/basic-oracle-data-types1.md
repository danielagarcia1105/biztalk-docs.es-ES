# <a name="basic-oracle-data-types"></a>Tipos de datos básicos de Oracle
Este tema se describe cómo el [!INCLUDE[adapteroracle](../../includes/adapteroracle-md.md)] expone tipos básicos de datos de Oracle.  

## <a name="supported-oracle-data-types"></a>Tipos de datos de Oracle compatibles  
 El [!INCLUDE[adapteroracle](../../includes/adapteroracle-md.md)] admite los tipos seguros para algunos tipos de datos de Oracle. Cuando escriba seguro está habilitado, estos tipos de datos se representan como cadenas. Configurar seguro escribiendo estableciendo el **EnableSafeTyping** enlaza la propiedad. Escribir seguro está deshabilitado de forma predeterminada. Para obtener más información sobre la [!INCLUDE[adapteroracle_short](../../includes/adapteroracle-short-md.md)] las propiedades de enlace, consulte [Obtenga información sobre las propiedades de enlace del adaptador de base de datos de Oracle](../../adapters-and-accelerators/adapter-oracle-database/read-about-the-oracle-database-adapter-binding-properties.md).  

> [!NOTE]
>  No se admite escribir seguro si son tipos de datos dentro de tipos definidos por el usuario (UDT).

### <a name="safe-typing-enabled"></a>Escribir seguro habilitado  
 En la tabla siguiente se muestra cómo se cambian los tipos de datos de Oracle que se ven afectados escribiendo seguro cuando el **EnableSafeTyping** enlaza la propiedad es true.  

|Tipo de datos de Oracle|Tipo XSD|Tipo de .NET|Comentario|  
|----------------------|--------------|---------------|-------------|  
|date|xsd:cadena|String|El valor se debería expresar en Oracle NLS_DATE_FORMAT.|  
|TimeStamp|xsd:cadena|String|El valor se debería expresar en Oracle NLS_TIMESTAMP_FORMAT.|  

> [!IMPORTANT]
>  Si escribe seguro está habilitado, los tipos de datos numéricos de Oracle dentro de los conjuntos de datos y débilmente tipada parámetros REF cursor siempre se exponen como cadenas.  

 Tipos de datos de Oracle que no están en esta tabla se exponen en la misma forma si escribe seguro está habilitado o deshabilitado.  

### <a name="validation"></a>Validación  
 El [!INCLUDE[adapteroracle](../../includes/adapteroracle-md.md)] no realiza ninguna validación explícita en los valores que especifique para los tipos de datos de Oracle. Sin embargo, según el tipo de datos de Oracle y si la escritura seguro está habilitado o deshabilitado, se puede realizar la validación implícita:  

-   Al deserializar entre en un mensaje y los tipos de .NET que se usan internamente por el adaptador pasa el XML.  

-   Por ODP.NET para algunos tipos de datos.
