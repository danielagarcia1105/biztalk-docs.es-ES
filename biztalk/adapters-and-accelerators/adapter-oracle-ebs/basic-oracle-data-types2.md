# <a name="basic-oracle-data-types"></a>Tipos de datos básicos de Oracle
Este tema se describe cómo el [!INCLUDE[adapteroracleebusinesslong](../../includes/adapteroracleebusinesslong-md.md)] expone tipos básicos de datos de Oracle.  

## <a name="supported-oracle-data-types"></a>Tipos de datos de Oracle compatibles  
 El [!INCLUDE[adapteroraclebusinessshort](../../includes/adapteroraclebusinessshort-md.md)] admite los tipos seguros para algunos tipos de datos de Oracle. Cuando escriba seguro está habilitado, estos tipos de datos se representan como cadenas. Configurar seguro escribiendo habilitando la **EnableSafeTyping** enlaza la propiedad (deshabilitado de forma predeterminada). Para obtener más información sobre la [!INCLUDE[adapteroraclebusinessshort](../../includes/adapteroraclebusinessshort-md.md)] las propiedades de enlace, consulte [Obtenga información sobre el adaptador de BizTalk para las propiedades de enlace de Oracle E-Business Suite](../../adapters-and-accelerators/adapter-oracle-ebs/read-about-the-biztalk-adapter-for-oracle-e-business-suite-binding-properties.md).

>  No se admite escribir seguro si son tipos de datos dentro de tipos definidos por el usuario (UDT).

## <a name="safe-typing-enabled"></a>Escribir seguro habilitado  
 En la tabla siguiente se muestra cómo se cambian los tipos de datos de Oracle que se ven afectados escribiendo seguro cuando el **EnableSafeTyping** es enlazar la propiedad **true**.  

> [!NOTE]
>  Tipos de datos de Oracle que no están en esta tabla se exponen en la misma forma si escribe seguro está habilitado o deshabilitado.  

|Tipo de datos de Oracle|Tipo XSD|Tipo de .NET|Comentario|  
|----------------------|--------------|---------------|-------------|  
|date|xsd:cadena|String|El valor se debería expresar en Oracle NLS_DATE_FORMAT. Puede especificar el formato para los tipos de datos de fecha en la **DateFormat** enlaza la propiedad en el **MlsSettings** enlaza la propiedad. Si se especifica ningún valor para el **DateFormat** propiedad de enlace, el adaptador utiliza los valores de MLS para el cliente ODP.NET en el mismo equipo donde está instalado el adaptador.|  
|TimeStamp|xsd:cadena|String|El valor se debería expresar en Oracle NLS_TIMESTAMP_FORMAT. Puede especificar el formato de cadena para tipos de datos de marca de tiempo en el **TimeStampFormat** enlaza la propiedad en el **MlsSettings** enlaza la propiedad. Si se especifica ningún valor para el **TimeStampFormat** propiedad de enlace, el adaptador utiliza los valores de MLS para el cliente ODP.NET en el mismo equipo donde está instalado el adaptador.|  

> [!IMPORTANT]
>  Si escribe seguro está habilitado, los tipos de datos numéricos de Oracle dentro de los conjuntos de datos y débilmente tipada parámetros REF cursor siempre se exponen como cadenas.  

## <a name="validation"></a>Validación  
 El [!INCLUDE[adapteroraclebusinessshort](../../includes/adapteroraclebusinessshort-md.md)] no realiza ninguna validación explícita en los valores que especifique para los tipos de datos de Oracle. Sin embargo, según el tipo de datos de Oracle y si la escritura seguro está habilitado o deshabilitado, se puede realizar la validación implícita:  

-   Al deserializar entre en un mensaje y los tipos de .NET que se usan internamente por el adaptador pasa el XML.  

-   Por ODP.NET para algunos tipos de datos.
