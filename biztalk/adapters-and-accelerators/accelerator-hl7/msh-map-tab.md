# <a name="msh-map-tab"></a>Pestaña asignación de MSH
Usa el **MSH mapa** ficha para habilitar las transformaciones de encabezado de mensaje para los mensajes entrantes.  
  
 En el **Explorador de configuración de BTAHL7** cuadro de diálogo la **MSH mapa** ficha, realice lo siguiente:  
  
|Use|Para|  
|--------------|----------------|  
|**MSH.3**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.4**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.5**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.6**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.8**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.9**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.12**|Campo de tipo de mensaje invalida para este encabezado de mensaje. **Nota:** para invalidar el valor existente en null, escriba  **\\** .|  
|**MSH.15**|Seleccione una de las siguientes opciones para un reemplazo de confirmación para el tipo de confirmación de aceptación:<br /><br /> -   **AL**. Seleccione esta opción si desea enviar siempre Aceptar confirmaciones.<br />-   **NE**. Seleccione esta opción si no desea enviar confirmaciones de Aceptar.<br />-   **SU**. Seleccione esta opción si desea enviar confirmaciones de Aceptar después de la transmisión correcta de un mensaje.<br />-   **ER**. Seleccione esta opción si desea enviar a Aceptar confirmaciones solo si se produce un error.|  
|**MSH.16**|Seleccione una de las siguientes opciones para un reemplazo de confirmación para el tipo de confirmación de la aplicación:<br /><br /> -   **AL**. Seleccione esta opción si desea enviar confirmaciones de aplicación siempre.<br />-   **NE**. Seleccione esta opción si no desea enviar confirmaciones de aplicación nunca.<br />-   **SU**. Seleccione esta opción si desea enviar confirmaciones de aplicación después de la transmisión correcta de un mensaje.<br />-   **ER**. Seleccione esta opción si desea enviar confirmaciones de aplicación sólo si se produce un error.|
